Chapter 15

비용 최적화 — ccusage 너머

"ccusage로 본다"는 측정일 뿐, 개선은 별개. 같은 작업을 1/3 비용으로 끝내는 6가지 패턴과 비용을 잡아먹는 4가지 안티패턴을 다룹니다.

먼저 진단 — 어떤 패턴이 비싼가

최적화 전에 측정. ccusage의 3가지 뷰를 차례로 봅니다.

npx ccusage@latest daily       # 일별: 어느 날이 비쌌는가
npx ccusage@latest session     # 세션별: 어떤 세션이 비쌌는가
npx ccusage@latest --order desc | head # 톱 비용 세션 즉시 확인

전형적 패턴 진단:

증상유력한 범인다음 장 액션
특정 세션 1개가 전체의 50%Opus + 큰 컨텍스트 장시간모델 분리 (§2)
매일 비슷한 청구액반복 작업에 캐싱 미적용프롬프트 캐싱 (§3)
input 토큰이 output의 20배+큰 파일·이미지 반복 첨부안티패턴 (§7)
cache_read 비율이 0%CLAUDE.md를 캐시로 안 씀캐싱 (§3)
CI/Action 비용 폭주모든 PR에 Opus + effort highCI 차등 (§6)

패턴 1 · 작업 유형별 모델 선택

가장 큰 절감 레버. Opus → Sonnet 다운그레이드만으로 3~5배 절감되는 작업이 많습니다.

작업 유형권장 모델비용 인덱스*
아키텍처 설계, 복잡 디버깅, 모호한 문제Opus 4.75
일반 기능 개발, 리팩터링, 코드 리뷰Sonnet 4.61
변환·분류·문서 생성, 대량 반복 작업Haiku 4.50.2
CI 자동 리뷰 (PR 1~5 파일)Sonnet 4.6 + effort medium0.6
CI 자동 리뷰 (대형 PR)Sonnet 4.6 + effort high1.2

*Sonnet 4.6 일반 사용 = 1 기준 상대값

세션 안에서 모델 전환

> /model                # 드롭다운에서 선택
> /effort medium         # 추론 깊이 조절 (low/medium/high/xhigh)
💡 한 세션, 두 모델

설계 단계는 Opus로, 그 다음 구현은 Sonnet으로 전환. 같은 세션에서 컨텍스트는 유지되면서 비용만 절감.

패턴 2 · 프롬프트 캐싱 활용

같은 시스템 프롬프트·CLAUDE.md·큰 문서를 반복 사용한다면 캐시된 토큰은 90% 저렴. Claude Code는 일정 조건에서 자동으로 캐싱하지만, 의식해서 도울 수 있습니다.

3가지 캐시 친화 습관

  1. CLAUDE.md는 자주 바꾸지 않기 — 매번 바뀌면 캐시 무효
  2. 큰 컨텍스트는 같은 세션에서 — 새 세션마다 다시 처음부터
  3. 5분 이내에 후속 질문 — 캐시 TTL 5분(기본), 활성 시 최대 1시간 연장

캐시 효율 측정

npx ccusage@latest daily --json | jq '.entries[] | {date, cache_read, input}'
# cache_read / (cache_read + input) 가 캐시 히트율
# 30% 넘으면 양호, 60% 넘으면 우수

패턴 3 · 서브에이전트로 비용 격리

"단순 작업은 Haiku 서브에이전트로"가 핵심 아이디어. 메인 컨텍스트(Opus·Sonnet)는 보존하고, 무거운 검색·파싱은 분리.

# .claude/agents/log-parser/agent.md
---
name: log-parser
description: "대량 로그·CSV 파싱"
model: claude-haiku-4-5-20251001  # ← Haiku 명시
tools: [Read, Bash, Grep]
---

대량 로그·텍스트 파싱 전문. 결과는 항상 요약된 표/JSON으로만 반환.

이후 메인 세션에서 "10MB 로그 분석" 같은 작업을 시키면, Claude가 이 서브에이전트에 위임 → 비용은 Haiku 가격 + 결과만 메인에 작은 형태로 돌아옴.

패턴 4 · /compact 적시 사용

/compact는 컨텍스트가 80% 넘을 때만 의미 있고, 그 전에 쓰면 오히려 손해(요약 자체에 토큰 소모).

컨텍스트 사용률액션
~60%그냥 진행
60~80%마무리 임박이면 그대로, 길게 갈 거면 /compact 한 번
80~95%"결론·다음 액션만 요약" → /compact
95%+새 세션으로 — 12장 참조

패턴 5 · Plan mode가 더 비쌀 때

Plan mode는 안전하지만 "계획 생성 + 실제 수정" 두 단계라 토큰이 1.3~1.5배. 큰 작업·위험 작업엔 가치 있지만, 작은 작업엔 사치.

작업 크기권장
한 파일 한 함수 수정Plan mode 끄기 — 그냥 진행
여러 파일·구조 변경Plan mode 켜기
위험 작업(결제·DB·인증)Plan mode 무조건 켜기

패턴 6 · CI 비용 차등화

모든 PR에 같은 강도로 리뷰하면 작은 PR도 큰 PR과 같은 돈을 씁니다. 변경 크기·민감도로 차등:

name: Claude Review
on: pull_request
jobs:
  light:
    if: github.event.pull_request.changed_files <= 5
    steps:
      - uses: anthropics/claude-code-action@v1
        with:
          prompt: "/code-review --effort low"
  full:
    if: github.event.pull_request.changed_files > 5
    steps:
      - uses: anthropics/claude-code-action@v1
        with:
          prompt: "/code-review --effort high"
  security:
    if: contains(github.event.pull_request.changed_files, 'auth/') || contains(github.event.pull_request.changed_files, 'payment/')
    steps:
      - uses: anthropics/claude-code-action@v1
        with:
          prompt: "/security-review --effort high"

비용 폭주 안티패턴 4가지

안티패턴토큰 영향대신
cat package-lock.json으로 통째로 보여주기 +50~200k "의존성 중 X 관련만 추출" 식 좁히기
스크린샷 4장 + "이거 분석해줘" +30~80k 한 장만 + 핵심 영역 크롭
같은 빌드 명령 5번 재시도 매 시도 += 20k 실패 패턴 분석부터, 무한 재시도 금지
find / -name "*.ts" (필터 없음) +10~100k find src -name "*.ts" -not -path "*/node_modules/*"

Pro/Max 한도 최대로 짜내기

API 직접 결제가 아니라 Pro/Max 구독이라면, 5시간 윈도우 안에서 최대 활용이 목표.

  • 가벼운 작업은 윈도우 시작 직후에 — 한도 절약
  • 긴 세션은 윈도우 후반 피하기 — 한도 직전에 중단되면 컨텍스트 손실
  • Usage Monitor (10장) 띄워 두고 70% 넘으면 모델 다운그레이드
  • 윈도우 리셋 시간 기억 — 한도 임박 시 "30분 뒤 시작이 더 효율적"

청구 알람·예산 설정

API 직접 결제 사용자 필수.

  • Anthropic Console: Settings → Billing → Spending limit 설정 (월 한도)
  • Email 알림: 한도의 50%·80%·100% 도달 시 자동 메일
  • 주간 자가 체크: /schedule "0 9 * * 1" sh -c 'npx ccusage weekly | slack-post'

5분 안에 적용하는 빠른 절감 5가지

  1. 모델 기본값을 Sonnet으로 — settings.json에 "model": "claude-sonnet-4-6"
  2. CLAUDE.md 200줄 이하로 다이어트 — 매 세션 절약
  3. 큰 파일 paste 금지 룰 — 본인에게도, 팀에게도 합의
  4. ccusage 주간 자동 리포트 — 가시화만 해도 30% 줄어듦
  5. CI에 effort 차등화 — 작은 PR엔 low

15장 체크리스트

  • ☐ 이번 주 ccusage 한 번 — 톱 비용 세션 패턴 식별
  • ☐ Sonnet을 기본 모델로 설정
  • ☐ CLAUDE.md를 200줄 이하로
  • ☐ 단순 작업용 Haiku 서브에이전트 1개 등록
  • ☐ CI에 변경 크기별 effort 차등 적용
  • ☐ Console에 월 spending limit 설정
  • ☐ 4가지 안티패턴 본인 작업에서 점검
💡 더 깊이 들어가고 싶다면 — 17장 사례집

이 장이 기법이라면 17장 · 비용 효율화 사례집은 이야기와 원칙입니다. 실제 70~80% 절감 사례 4개, 프로젝트 단계별 효율 가이드, Claude에 국한되지 않는 AI 활용 일반 원칙 8가지까지 다룹니다.
