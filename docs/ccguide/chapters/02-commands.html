Chapter 02

슬래시 명령·권한·메모리·훅

Claude Code의 "조종 장치" 네 가지를 한 번에 익힙니다. 슬래시 명령어로 세션을 통제하고, 권한으로 안전을 보장하며, CLAUDE.md로 프로젝트 컨텍스트를 영구 학습시키고, 훅으로 자동 동작을 박아 둡니다.

자주 쓰는 슬래시 명령어

전부 외울 필요는 없습니다. /help로 언제든 목록을 볼 수 있어요. 다만 아래 표는 매주 만나는 명령들입니다.

세션 관리

명령동작
/help사용 가능한 모든 명령 목록
/clear현재 대화 히스토리 비우기 (CLAUDE.md는 유지)
/compact대화 컨텍스트 압축 (긴 대화 지속 가능)
/context현재 토큰 사용량 확인
/resume과거 세션 목록에서 골라 이어가기
/exit 또는 Ctrl+D세션 종료

설정·메모리·도구

명령동작
/config설정 GUI (모델·테마·언어 등)
/model모델 변경 (Opus / Sonnet / Haiku)
/permissions권한 규칙 추가/조회
/initCLAUDE.md 자동 생성 / 갱신
/memoryCLAUDE.md · 자동 메모리 열람/편집
/agents서브에이전트 관리
/mcpMCP 서버 추가/조회/인증

/doctor설정 진단 (문제 발생 시 1순위)
💡 팁 — 명령어 자동완성

슬래시(/)만 치면 사용 가능한 명령이 드롭다운으로 떠요. 스킬(3장)과 플러그인도 같은 자리에 노출됩니다. 실제로 외울 명령은 위 표 정도면 충분합니다.

🎬 시나리오 · "처음 들어간 프로젝트 (/init)"
상황: 신입으로 입사 첫날. 50개 디렉토리, 수백 파일의 모놀리식 프로젝트. 어디부터 봐야 할지 모르겠음. 이전 방식: README 정독 → 핵심 파일 찾아 헤매기 → 1시간 이상. 적용: 
cd ~/projects/big-mono
claude
> /init
# Claude가 코드를 훑고 CLAUDE.md 초안 작성
# → 빌드/테스트 명령, 모듈 구조, 의존성 흐름 자동 정리

> "CLAUDE.md에 적힌 구조 중 가장 중요한 모듈 3개를 골라 각각 한 문단으로 설명"
결과: 15분 만에 프로젝트 지도를 손에 쥠. CLAUDE.md는 그대로 커밋해 다음 사람도 같은 길로.
🎬 시나리오 · "긴 디버깅 도중 컨텍스트 꽉 참 (/compact)"
상황: 3시간째 같은 버그 추적. 대화가 100메시지를 넘어 응답이 점점 느려지고, "이전에 시도한 것"을 Claude가 잊기 시작. 이전 방식: /clear → 처음부터 다시 설명. 컨텍스트 다 날아감. 적용: 
> /context              # 현재 사용량 확인 (예: 180k/200k)
> /compact              # 핵심 결론만 요약하고 나머지 압축
> "지금까지 시도한 가설 3개와 가장 유력한 원인 1개를 다시 요약해줘"
결과: 핵심 컨텍스트는 보존된 채 60k로 다이어트. 디버깅 흐름 끊기지 않고 계속.
🎬 시나리오 · "팀에 새 사람 합류 (CLAUDE.md)"
상황: 신입에게 매번 "우리는 2-space 인덴트", "lint 돌리고 커밋", "core 폴더는 함부로 건드리지 마" 반복 설명. 적용: 같은 내용을 CLAUDE.md에 박아 두면 본인뿐 아니라 모든 팀원의 Claude가 자동 학습. 
# CLAUDE.md (커밋 대상)
## Code Standards
- 2-space indent, camelCase
- Always run `npm run lint` before commit

## Common Mistakes
- Do NOT modify `src/core/engine.ts` without integration tests
- Never commit `.env`
결과: 신입의 첫 PR이 처음부터 팀 컨벤션과 맞아 들어옴. 리뷰 코멘트 80% 감소.

권한 시스템 — Deny > Ask > Allow

Claude Code의 권한은 3단계 규칙으로 동작합니다. 우선순위는 Deny > Ask > Allow 순. 한 번 학습해 두면 평생 씁니다.

3가지 규칙

규칙동작예시
Allow프롬프트 없이 자동 실행"Bash(npm run test *)"
Ask매번 실행 전 확인"Edit(.env)"
Deny아예 실행 금지"Bash(rm -rf *)"

설정 위치 — .claude/settings.json

{
  "defaultMode": "plan",
  "permissions": {
    "allow": [
      "Bash(npm run test *)",
      "Bash(git commit *)",
      "Bash(git log *)",
      "Read(.env)"
    ],
    "ask": [
      "Bash(rm *)",
      "Edit(src/core/**)"
    ],
    "deny": [
      "Bash(curl *)",
      "WebFetch(domain:example.com)",
      "Read(~/.ssh/**)"
    ]
  }
}

패턴 문법 핵심

  • Bash(npm run *) — npm run 뒤에 무엇이 와도 매칭
  • Edit(src/**/*.ts) — glob 패턴으로 경로 제한
  • WebFetch(domain:github.com) — 도메인 단위 제어
  • mcp__github__* — MCP 서버의 특정 도구만/전부 제어

권한 프롬프트 줄이는 4단계

  1. 실시간으로 "Yes, don't ask again" 선택 — 가장 빠른 방법.
  2. /permissions GUI로 직접 추가.
  3. .claude/settings.json 편집으로 팀 단위 공유.
  4. /fewer-permission-prompts 스킬 — 지난 세션을 분석해 자주 쓴 명령을 allow 후보로 자동 추천 (3장).
🎓 강사 노트 — Deny는 작게, Allow는 천천히

"안전하니까 Deny를 잔뜩 걸자"는 매력적이지만, 실제로는 Deny는 진짜 위험한 것만 등록하고 Allow는 작업하면서 천천히 늘려 가는 편이 좋습니다. Deny가 너무 넓으면 Claude가 작은 작업도 못 해서 의외로 답답해집니다. 대신 Plan mode를 함께 켜 두면 Deny 없이도 큰 사고를 막을 수 있어요.

Permission Modes (전역 모드)

위 3규칙과 별개로, 세션 전체의 동작 모드도 바꿀 수 있습니다.

모드동작사용 시기
default처음 한 번만 묻고 기억일상 개발
plan모든 수정 전에 계획 제시 → 승인 요청신중한 검토 필요
acceptEdits파일 편집을 자동 승인잘 아는 정형 작업
bypassPermissions거의 모든 권한 자동 (위험)격리된 VM·컨테이너 안에서만

settings.json — 어디에 무엇이 들어가나

설정 파일은 4계층입니다. 위로 갈수록 우선순위가 높습니다.

레이어경로공유 여부용도
관리자 정책OS별 시스템 경로강제회사 IT 정책
로컬(개인 프로젝트).claude/settings.local.jsonX (gitignore)나만의 모델·테마·API 키
프로젝트(팀 공유).claude/settings.jsonO (커밋)팀 공통 권한·훅·언어 설정
사용자(전역)~/.claude/settings.jsonX모든 프로젝트 공통

자주 쓰는 키

{
  // 모델 · 추론
  "model": "claude-sonnet-4-6",
  "effortLevel": "high",

  // 권한
  "defaultMode": "plan",
  "permissions": { "allow": ["Bash(npm run *)"], "deny": ["Bash(rm -rf *)"] },

  // UX
  "theme": "dark",
  "language": "korean",
  "editorMode": "vim",

  // 메모리·컨텍스트
  "autoMemoryEnabled": true,
  "autoCompactEnabled": true,

  // 환경변수 (모든 도구 실행에 주입)
  "env": { "NODE_ENV": "development" }
}
⚠️ 주의 — 비밀은 절대 settings.json에

API 키·토큰은 .claude/settings.json (커밋 대상)에 넣으면 큰 일 납니다. 반드시 .claude/settings.local.json 또는 OS 환경변수로.

CLAUDE.md — 프로젝트 영구 컨텍스트

CLAUDE.md는 매 세션 자동으로 읽히는 지시사항 파일입니다. 잘 쓰면 같은 설명을 반복할 필요가 없어집니다.

종류

위치범위커밋?
~/.claude/CLAUDE.md모든 프로젝트 (개인 전역)X
./CLAUDE.md이 프로젝트 (팀 공유)O
./CLAUDE.local.md이 프로젝트 (개인용)X (gitignore)

잘 쓴 CLAUDE.md 예시

# Backend API Project

## Quick Start
- Install: `npm install`
- Dev: `npm run dev` (http://localhost:3000)
- Test: `npm run test:watch`
- Build: `npm run build`

## Architecture
- Entry: `src/index.ts`
- Routes: `src/routes/`
- Models: `src/models/` (Mongoose schemas)

## Code Standards
- Indentation: 2 spaces
- camelCase for functions, PascalCase for classes
- TypeScript types required
- Run `npm run lint` before committing

## Common Mistakes (do NOT)
- Do NOT modify `src/core/engine.ts` without running integration tests
- Do NOT commit `.env`

작성 원칙 5가지

  1. 200줄 이하를 목표 — 매 세션마다 토큰 소모. 길수록 Claude가 무시할 가능성도 증가.
  2. 구체성 — "좋은 포맷"이 아니라 "2-space 인덴트"처럼 검증 가능하게.
  3. 해야 할 것 + 하지 말 것을 같이 명시.
  4. 큰 파일은 .claude/rules/*.md로 분리하고 @로 임포트.
  5. /init으로 초안 자동 생성 후 다듬는 게 가장 빠릅니다.
📌 참고 — Auto Memory

Claude는 자기 자신을 위한 자동 메모리도 따로 적습니다(~/.claude/projects/<proj>/memory/MEMORY.md). 세션마다 "사용자가 ~를 선호함", "이 빌드 명령이 맞음" 등을 누적 학습합니다. /memory로 열람·정리 가능합니다.

훅(Hooks) — 자동 동작 박아 넣기

훅은 특정 이벤트가 발생할 때 자동 실행되는 쉘 명령입니다. 권한과 함께 "안전·자동화"의 양 축입니다.

주요 이벤트

이벤트시점흔한 용도
PreToolUse도구 사용 직전커스텀 검증·차단
PostToolUse도구 실행 후로깅·후처리
FilesModified파일 수정 후자동 포맷팅·린트
SessionStart세션 시작환경 점검·환영 메시지
SessionEnd세션 종료요약 저장·정리

설정 예시 — 저장 후 자동 포맷

{
  "hooks": [
    {
      "event": "FilesModified",
      "command": "npx prettier --write $INPUT_paths"
    },
    {
      "event": "PreToolUse",
      "tool": "Bash",
      "command": "~/.claude/hooks/safe-bash.sh"
    }
  ]
}

쉘 스크립트 예 — 위험 명령 차단

#!/bin/bash
# ~/.claude/hooks/safe-bash.sh
if [[ "$INPUT_command" == *"rm -rf /"* ]]; then
  echo "BLOCKED: dangerous rm -rf detected"
  exit 1  # non-zero → 도구 실행 차단
fi
exit 0
🎓 강사 노트 — 훅 vs 권한, 무엇을 쓸까

둘은 보완 관계입니다. 권한은 정적 규칙(JSON)으로 빠르고 명확합니다. 훅은 동적 검증(쉘 스크립트)이 가능해 더 유연하지만 디버깅이 까다롭습니다. 간단한 allow/deny는 권한으로, "명령어 안 인자까지 보고 결정"이 필요한 케이스만 훅으로 가는 게 깔끔합니다.

2장 체크리스트

  • ☐ /help로 명령 목록 한 번 훑기
  • ☐ .claude/settings.json에 권한 allow 3~5개 등록
  • ☐ defaultMode를 plan으로 설정
  • ☐ CLAUDE.md를 200줄 이내로 정리
  • ☐ (선택) 저장 후 자동 포맷 훅 1개 등록
