Claude API — 모델을 직접 호출하기

Claude Code·Agent SDK 너머 — 모델을 가장 낮은 수준에서 직접 호출하는 방법. 프롬프트 캐싱으로 비용 90% 절감, Extended Thinking으로 복잡 문제 추론, Tool Use·Batch·Files API까지.

최신 모델 ID와 선택 트리

모델모델 ID최적 용도1M 컨텍스트
Opus 4.7claude-opus-4-7복잡한 추론·설계·창의적 합성✅ claude-opus-4-7[1m]
Sonnet 4.6claude-sonnet-4-6일반 코딩·균형·기본값✅
Haiku 4.5claude-haiku-4-5-20251001분류·변환·고속 처리✅
💡 선택 트리 (15장 표 요약)
  • 설계·복잡 디버깅·모호한 문제 → Opus 4.7
  • 일반 코딩·리뷰·리팩터링 → Sonnet 4.6 (디폴트)
  • 대량 분류·변환·요약 → Haiku 4.5
  • 200k 초과 컨텍스트 필요 → 모델 ID 뒤에 [1m] 변형

const stream = await client.messages.create({
  model: "claude-sonnet-4-6",
  max_tokens: 1024,
  stream: true,
  messages: [{ role: "user", content: "한국 역사 삼국시대 5줄" }],
});

for await (const event of stream) {
  if (event.type === "content_block_delta" && event.delta.type === "text_delta") {
    process.stdout.write(event.delta.text);
  }
}

프롬프트 캐싱 — 비용 90% 절감의 진짜 무기

같은 시스템 프롬프트·문서를 여러 번 호출한다면 무조건 활성화. 캐시 토큰은 입력 토큰의 10% 비용이고 응답도 더 빠릅니다.

언제 캐싱이 본전 뽑히는가

  • 같은 시스템 프롬프트로 N번 호출 (챗봇·고객 지원)
  • 큰 문서(API 스펙·매뉴얼)를 컨텍스트로 두고 반복 질의
  • 대화형 세션 (이전 대화를 매번 보냄)

활성화 — cache_control 한 줄

large_doc = open("api-spec.md").read()  # 50KB 문서

response = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=1024,
    system=[
        {"type": "text", "text": "You are a senior API designer."},
        {
            "type": "text",
            "text": large_doc,
            "cache_control": {"type": "ephemeral"},  // ← 캐싱 켜기
        },
    ],
    messages=[{"role": "user", "content": "이 API에서 누락된 인증 케이스 3가지"}],
)
print(f"cache_creation: {response.usage.cache_creation_input_tokens}")
print(f"cache_read    : {response.usage.cache_read_input_tokens}")
# 두 번째 요청부터 cache_read가 대부분 차지하면 성공
import fs from "node:fs";
const largeDoc = fs.readFileSync("api-spec.md", "utf8");

const response = await client.messages.create({
  model: "claude-sonnet-4-6",
  max_tokens: 1024,
  system: [
    { type: "text", text: "You are a senior API designer." },
    { type: "text", text: largeDoc, cache_control: { type: "ephemeral" } },
  ],
  messages: [{ role: "user", content: "누락된 인증 케이스 3가지" }],
});

console.log(`cache_creation: ${response.usage.cache_creation_input_tokens}`);
console.log(`cache_read    : ${response.usage.cache_read_input_tokens}`);
🎓 강사 노트 — TTL과 효율

캐시 TTL은 기본 5분, 활성 사용 시 최대 1시간까지 연장. 5분 이내 후속 요청에서 본전이 빠지므로, 대화형 봇이라면 사용자 응답 직후 다음 메시지가 빨리 들어와야 좋습니다. 배치 처리는 같은 시스템을 묶어서 연속 호출해야 캐시 효과가 큽니다.

Extended Thinking — 복잡 추론에 더 깊은 사고

모델이 응답 전 "생각하는 단계"를 별도 토큰 예산으로 할당. 수학·논리·다단계 디버깅에 유리.

response = client.messages.create(
    model="claude-opus-4-7",
    max_tokens=16000,           # thinking + 최종 답 모두 포함
    thinking={
        "type": "enabled",
        "budget_tokens": 10000   # 사고에 쓸 토큰
    },
    messages=[{"role": "user", "content":
        "이 코드의 race condition을 단계별로 분석하고, 가능한 시나리오 3가지 제시"}],
)

for block in response.content:
    if block.type == "thinking":
        print("=== 사고 과정 ===", block.thinking)
    elif block.type == "text":
        print("=== 최종 답 ===", block.text)
⚠️ Thinking 토큰도 비용

budget_tokens는 응답 토큰 단가로 청구됩니다. 단순 작업엔 끄세요(또는 budget을 낮게). 반대로 정말 복잡한 문제엔 후하게 — 잘못된 답을 디버깅하는 시간이 더 비쌉니다.

Tool Use — 함수 호출 (Function Calling)

SDK 안 쓰고 API 레벨에서 직접 도구 호출 흐름을 다루기. 일반적으로 SDK가 편하지만, 흐름 이해는 필수.

tools = [{
    "name": "get_weather",
    "description": "도시의 현재 날씨 조회",
    "input_schema": {
        "type": "object",
        "properties": {"city": {"type": "string"}},
        "required": ["city"]
    }
}]

messages = [{"role": "user", "content": "서울이랑 부산 날씨 알려줘"}]
response = client.messages.create(model="claude-sonnet-4-6", max_tokens=1024, tools=tools, messages=messages)

# 에이전트 루프
while response.stop_reason == "tool_use":
    tool_calls = [b for b in response.content if b.type == "tool_use"]
    tool_results = []
    for call in tool_calls:
        result = my_get_weather(call.input["city"])  # 실제 함수 실행
        tool_results.append({"type": "tool_result", "tool_use_id": call.id, "content": result})

    messages.append({"role": "assistant", "content": response.content})
    messages.append({"role": "user",      "content": tool_results})
    response = client.messages.create(model="claude-sonnet-4-6", max_tokens=1024, tools=tools, messages=messages)

print("".join(b.text for b in response.content if b.type == "text"))

MCP를 API 클라이언트로 사용

4장에서 본 MCP를 API에서도 직접 연결 가능. SDK가 자동 처리해 주지만, 헤더로 명시할 수도 있습니다.

# mcp_servers 파라미터에 등록
response = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=1024,
    mcp_servers=[{
        "type": "url",
        "url": "https://mcp.sentry.dev/mcp",
        "name": "sentry",
        "authorization_token": os.environ["SENTRY_TOKEN"],
    }],
    messages=[{"role": "user", "content": "지난 1시간 에러 톱 5"}],

Files API — PDF·이미지·문서 전송

큰 파일은 미리 업로드해 두고 file_id로 참조 → 매번 base64 인코딩 부담 제거.

# 1) 업로드 (한 번만)
upload = client.files.create(file=open("manual.pdf", "rb"))
print(upload.id)  # file_xxxxx

# 2) 메시지에서 참조
response = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=1024,
    messages=[{
        "role": "user",
        "content": [
            {"type": "document", "source": {"type": "file", "file_id": upload.id}},

Batch API — 비동기 대량 처리 50% 절감

"지금 당장" 필요 없는 작업(분류·요약 대량)은 Batch로 보내면 비용 절반. 처리 시간은 보통 1시간 이내(SLA 최대 24h).

# 1) 배치 요청 준비
requests = [
    {
        "custom_id": f"feedback-{i}",
        "params": {
            "model": "claude-haiku-4-5-20251001",
            "max_tokens": 256,
            "messages": [{"role": "user", "content": f"분류: {text}"}],
        }
    }
    for i, text in enumerate(feedbacks)
]

# 2) 배치 제출
batch = client.messages.batches.create(requests=requests)
print(f"batch_id: {batch.id}")

# 3) 폴링 (또는 webhook으로 받기)
while batch.processing_status != "ended":
    time.sleep(60)
    batch = client.messages.batches.retrieve(batch.id)

# 4) 결과 가져오기
for result in client.messages.batches.results(batch.id):
    print(result.custom_id, result.result.message.content[0].text)

API 키·청구 안전

  • 키 분리: dev/staging/prod 별도 키 → 사고 시 회전 쉬움
  • 키 환경변수만: 코드·CLAUDE.md·Git에 키 박지 말 것
  • Spending limit: Anthropic Console → Billing → 월 한도 + 50/80/100% 알림
  • 로그: 모든 API 호출의 request_id·model·tokens 기록 (감사용)
  • 레이트 리밋: 429 받으면 지수 백오프 (SDK는 자동, raw cURL은 직접)

8장 체크리스트

  • ☐ 본인 작업의 적정 모델 선정 (Opus/Sonnet/Haiku)
  • ☐ 반복 호출이라면 프롬프트 캐싱 활성화
  • ☐ 대량 처리는 Batch API 검토 (50% 절감)
  • ☐ 큰 파일은 Files API로 업로드 후 file_id 참조
  • ☐ API 키 환경변수 + dev/prod 분리
  • ☐ Console에 spending limit 설정
  • ☐ 7장 SDK와 비교 후 알맞은 추상화 수준 선택
