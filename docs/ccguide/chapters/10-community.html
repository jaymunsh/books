커뮤니티 스킬·툴 베스트 — 2026년 핫한 것들

Claude Code 생태계는 폭발적으로 자라고 있습니다. 이 장에서는 GitHub 별이 빠르게 오른 커뮤니티 도구들을 골라 "언제·어떻게 쓰면 되는지"의 시나리오와 함께 소개합니다. 별 수치는 2026년 6월 기준입니다(빠르게 변하니 참고용).

📌 어떻게 골랐나

① GitHub 별 1k+ ② 최근 6개월 안에 활발히 업데이트 ③ 실무에서 실제 쓸 만한 시나리오가 있는 것 — 세 조건을 만족하는 것들만 추렸습니다. 커뮤니티 프로젝트는 빠르게 변하므로, 설치 전 반드시 저장소의 최신 README를 확인하세요.

⭐ superpowers — TDD 강제 풀스택 SDLC 프레임워크

obra/superpowers ★ 233k

🔗 github.com/obra/superpowers

커뮤니티에서 가장 큰 Claude Code 스킬 라이브러리. 브레인스토밍 → 스펙 → 계획 → 서브에이전트 분담 → TDD → 리뷰 → 머지까지 소프트웨어 개발 전체 사이클을 스킬 묶음으로 강제합니다. 핵심 철학은 RED → GREEN → REFACTOR 사이클을 코드 작성 매 단계마다 따르게 하는 것.

설치 (Claude Code 플러그인):

/plugin install superpowers@claude-plugins-official
🎬 시나리오 · "기능 하나를 처음부터 끝까지 맡기기"
상황: 결제 API 엔드포인트 1개를 새로 만들어야 하는데, 매번 빠뜨리는 단계(테스트 작성·엣지 케이스·문서)가 있어 머지 후에 후회한다. 이전 방식: "결제 API 만들어줘" → Claude가 코드만 후딱 짜고 끝. 테스트는 따로 시켜야 함. superpowers 적용: 
> /superpowers:plan-feature "POST /payments — 카드 결제 생성"
# 1) 스펙 초안 자동 생성 (요청·응답·에러 케이스)
# 2) 실패하는 테스트 먼저 작성 (RED)
# 3) 통과시키는 최소 구현 (GREEN)
# 4) 리팩터링 → 보안·성능 리뷰
# 5) PR 본문·체인지로그 자동 생성
결과: 시간은 30% 정도 더 들지만, 머지 직전 "테스트·문서 빠졌네" 0건. 큰 작업일수록 효과가 큽니다.

⭐ awesome-agent-skills — 1000+ 스킬 큐레이션

VoltAgent/awesome-agent-skills ★ 25.8k

🔗 github.com/VoltAgent/awesome-agent-skills

공식 팀과 커뮤니티가 만든 1000개 이상의 에이전트 스킬을 카테고리별로 정리. Claude Code뿐 아니라 Codex·Gemini CLI·Cursor 와도 호환되는 포맷이라 스킬 자체의 이식성이 좋습니다. 인프라·실행·데이터 접근·코드 품질·오케스트레이션 5개 영역으로 분류돼 있어 필요한 것만 골라 쓸 수 있어요.

🎬 시나리오 · "스킬을 직접 만들기 전에 한 번 둘러보기"
상황: "릴리즈 자동화 스킬을 만들까…" 하다가 시간 낭비. 적용: awesome-agent-skills에서 release·changelog 검색 → 유사 스킬 3~4개 README 비교 → 가장 잘 맞는 것 골라 .claude/skills/에 복사 → 우리 팀 컨벤션만 살짝 수정. 결과: 1시간 작업이 15분으로. 처음부터 안 만드는 게 잘 만드는 첫 단계입니다.

⭐ awesome-claude-code-toolkit — 올인원 키트

🔗 github.com/rohitg00/awesome-claude-code-toolkit

혼자 다 짜기 부담스러운 사람을 위한 올인원 패키지: 135개 에이전트, 35개 큐레이션 스킬, 42개 커스텀 명령, 176+ 플러그인, 20개 훅, 15개 룰, 14개 MCP 설정까지. 카테고리는 개발·DevOps·QA·데이터 과학·비즈니스 등 11개로 나뉘어 있어요.

설치 (플러그인 마켓플레이스):

/plugin marketplace add rohitg00/awesome-claude-code-toolkit
🎬 시나리오 · "처음 Claude Code 도입하는 팀"
상황: 회사에 Claude Code를 도입했는데, 팀원마다 사용법이 제각각이고 셋업도 비효율. 적용: 신규 도입 첫날 /plugin marketplace add ...로 전체 키트를 깔고, 우리 팀이 안 쓸 것은 비활성화. 1주일간 모두가 같은 스킬·훅을 공유하며 일관성 확보. 결과: "이런 작업 어떻게 해요?"의 답이 "스킬 /xxx 써보세요"로 통일. 온보딩 시간 절반 이상 단축.

⭐ ccusage — 토큰·비용 모니터링 표준

ryoppippi/ccusage ★ 16.4k

🔗 github.com/ryoppippi/ccusage

로컬에 쌓이는 Claude Code의 .jsonl 사용 로그를 분석해 일/주/월/세션별 토큰·비용 리포트를 표로 출력합니다. 5시간 청구 윈도우·모델별 분리·JSON 출력·타임존 지원까지. 사실상 커뮤니티 표준 트래커입니다.

설치 없이 한 줄 실행:

npx ccusage@latest             # 기본 일별 리포트
npx ccusage@latest monthly     # 월별 합계
npx ccusage@latest session     # 세션별
npx ccusage@latest --json      # JSON 출력 (자동화용)
🎬 시나리오 1 · "이번 달 얼마 썼지?"
상황: API 크레딧으로 결제 중인데, 월말에 "왜 이렇게 많이 나왔지?" 매번 놀란다. 적용: 매주 금요일 퇴근 전에 한 줄. 
npx ccusage@latest weekly
결과: 비싼 작업(주로 Opus + 큰 컨텍스트)이 어디에 몰렸는지 표로 보임. 다음 주엔 그 작업만 Sonnet으로.
🎬 시나리오 2 · "팀 사용량 대시보드"
적용: /schedule로 매일 새벽 ccusage 결과를 슬랙으로. 
> /schedule "0 7 * * *" sh -c 'npx ccusage@latest daily --json | jq ... | slack-post #ops-claude'

⭐ Claude Code Usage Monitor — 실시간 터미널 게이지

Maciek-roboblog/Claude-Code-Usage-Monitor

🔗 github.com/Maciek-roboblog/Claude-Code-Usage-Monitor

ccusage가 리포트형이라면 이건 실시간 게이지형입니다. 별도 터미널에 띄워 두면 현재 5시간 윈도우 사용량·잔여 토큰·ML 기반 예측이 계속 갱신됩니다. 프로 플랜의 윈도우 제한을 자주 만나는 헤비 유저용.

🎬 시나리오 · "마감 직전 토큰 폭주 방지"
상황: 큰 리팩터링 중간에 갑자기 "토큰 한도 초과 — 2시간 대기"가 떠 작업이 막힘. 적용: 모니터를 2번째 모니터에 띄워 두고, 사용률 70% 넘으면 Opus → Sonnet으로 다운그레이드.

⭐ claude-code-action — 공식 GitHub Action

anthropics/claude-code-action 공식

🔗 github.com/anthropics/claude-code-action

Anthropic이 공식 배포하는 GitHub Action. PR이 열리면 자동 리뷰하거나, 이슈/PR 코멘트에 @claude를 멘션하면 브랜치 생성 → 수정 → 테스트 → PR까지 자동으로. 50 PR/월 정도면 비용은 보통 $5 안쪽으로 알려져 있습니다.

최소 설정 (.github/workflows/claude.yml):

name: Claude
on:
  issue_comment: { types: [created] }
  pull_request_review_comment: { types: [created] }

jobs:
  claude:
    if: contains(github.event.comment.body, '@claude')
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: anthropics/claude-code-action@v1
        with:
          anthropic_api_key: ${{ secrets.ANTHROPIC_API_KEY }}
🎬 시나리오 · "이슈 던지면 PR이 돌아온다"
상황: 작은 버그가 잔뜩. 매번 직접 브랜치 만들고 고치기 귀찮음. 적용: 이슈 본문에 재현 절차 적고 코멘트로 @claude 이거 고쳐줘. 결과: 10분 뒤 봇이 브랜치 만들고 수정 후 PR 올림. 나는 리뷰만 누름. 작은 버그 처리 속도가 5배.
⚠️ 주의 — 외부 PR에는 신중히

외부 컨트리뷰터의 PR에 자동 리뷰가 트리거되면 비용·악용 위험이 있습니다. if 조건에 github.event.pull_request.author_association 체크를 꼭 넣으세요.

🎓 강사 노트 — 커뮤니티 도구를 깔 때 5가지 체크
  1. 최근 커밋 날짜 — 3개월 이상 멈춰 있으면 패스
  2. 이슈 응답 속도 — 메인테이너가 살아 있는지 확인
  3. README의 보안 섹션 — 어떤 권한이 필요한지 명시돼 있는가
  4. 스킬 본문 검토 — 깔기 전에 SKILL.md를 직접 읽고, 위험한 명령이 들어 있지 않은지 확인
  5. 본인 settings.json의 deny와 충돌 — 깐 뒤에도 deny 규칙은 그대로 유지하기

10장 체크리스트

  • ☐ npx ccusage@latest 한 번 실행해 본인 사용량 확인
  • ☐ 위 5가지 안전 체크를 통과한 커뮤니티 스킬 1개 시범 도입
  • ☐ (Github 리포 사용자) claude-code-action 워크플로우 1개 등록
  • ☐ 우리 팀에 필요 없는 스킬은 비활성화 — 명령 자동완성이 깔끔해짐
  • ☐ awesome-* 리스트 1개 북마크해 두기 (도구 떠오를 때 첫 검색지)
