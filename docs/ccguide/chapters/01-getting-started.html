Chapter 01

시작하기 — 설치부터 첫 세션까지

이 장은 Claude Code를 처음 켜는 순간부터 첫 코드를 수정해 보기까지를 다룹니다. 30분이면 끝나며, 이 장만 마쳐도 "평소처럼 코딩하다 막힐 때 도움받기"는 바로 시작할 수 있습니다.

설치 및 인증

Claude Code는 터미널에서 실행되는 AI 코딩 에이전트입니다. macOS·Linux·Windows(WSL2 권장)에서 동작합니다.

# 공식 설치 스크립트
curl -fsSL https://claude.ai/install.sh | bash

# 설치 확인
claude --version
# Homebrew Cask
brew install --cask claude-code

# 업데이트
brew upgrade claude-code
# PowerShell (관리자 권한 권장)
irm https://claude.ai/install.ps1 | iex

# 설치 확인
claude --version

로그인

최초 실행 시 자동으로 브라우저 로그인 창이 뜹니다. 세션 안에서는 /login으로 다시 로그인할 수 있어요.

# 첫 실행 → 브라우저 자동 오픈 → 로그인
claude

# 세션 안에서 재로그인
/login

지원 계정은 다음과 같습니다.

  • Claude Pro / Max / Team / Enterprise (가장 일반적)
  • Anthropic Console (API 크레딧으로 사용)
  • Amazon Bedrock · Google Vertex AI · Microsoft Foundry (기업/규제 환경)
💡 팁 — 처음엔 Pro/Max 권장

API 크레딧 결제 방식은 토큰량을 매번 신경 써야 합니다. 학습 단계에서는 Pro 또는 Max 구독이 마음 편합니다. 팀 단위라면 Team/Enterprise로 청구·정책 관리를 한곳에 묶을 수 있습니다.

실행 모드 — 한 번에 정리

Claude Code는 같은 claude 명령에 옵션을 붙여 여러 모드로 동작합니다. 한 표로 외우세요.

모드실행 방법언제 쓰는가
Interactive (대화형)claude가장 일반적. 대화하면서 계속 작업
One-shotclaude "질문/작업"한 번만 실행하고 종료. 스크립트에서 호출
Printclaude -p "질문"상호작용 없이 결과만 표준출력. 파이프·CI에 적합
Continueclaude -c가장 최근 세션 그대로 이어가기
Resumeclaude -r 또는 /resume여러 과거 세션 중 선택해 이어가기
Plan mode세션 중 Shift+Tab 토글 또는 /plan실제 수정 전에 계획만 받고 승인 → 안전
Fast mode/fastOpus 4.6 / 4.7에서 빠른 응답 모드
Headlessclaude -p ... && ...CI/CD·크론·서버에서 비대화식 실행

실제 사용 예

# 평소 작업 — 그냥 진입
cd ~/my-project
claude

# "이 README를 200자로 요약해줘" → 결과만 stdout
claude -p "README.md를 200자 한국어로 요약"

# 어제 하던 대화 그대로 이어가기
claude -c

# 새 기능을 격리된 워크트리에서 시작
claude --worktree feature/auth
🎓 강사 노트 — Plan mode를 디폴트로 두세요

처음 한 달은 무조건 Plan mode로 쓰세요. 큰 변경 전에 Claude가 계획을 먼저 보여주고, 당신이 승인해야 실행됩니다. 익숙해지면 Shift+Tab으로 토글해서 평소엔 일반 모드, 위험 작업만 plan으로 가는 식으로 운영하면 됩니다.

🎬 시나리오 · "슬랙에서 빠른 질문 받았을 때 (one-shot)"
상황: 동료가 슬랙으로 "이 에러 메시지 뭐야?" 스크린샷을 보냈음. 세션 켜고 대화하긴 과함. 적용: 터미널 한 줄. 
claude -p "이 Node.js 에러 메시지가 무슨 뜻인지 한 줄로: TypeError: Cannot read properties of undefined (reading 'map')"
결과: 답변이 stdout으로만 나옴 → 그대로 복사해 슬랙에 붙여넣기. 30초 컷.
🎬 시나리오 · "처음 만지는 큰 변경, 무서워서 Plan mode"
상황: 인증 미들웨어 전체를 갈아엎어야 함. Claude가 곧장 코드 갈아엎어 버리면 무서움. 이전 방식: 작업 시키고 매번 "잠깐, 그건 아냐" 끊기 반복. 적용: 세션 들어가서 Shift+Tab → Plan mode로 전환. "auth 미들웨어를 JWT 기반으로 교체해줘"라고 요청. 결과: Claude가 코드 안 건드리고 "이런 파일을 이렇게 바꾸겠다"는 계획만 제시 → 당신이 검토 후 승인 → 그때부터 실제 수정. 안전 + 빠름.
🎬 시나리오 · "메인 브랜치 만지지 않고 실험 (worktree)"
상황: 현재 다른 작업 중인데, 누가 "이거 한번만 빠르게 시도해줄래?" 적용: 메인 디렉토리는 그대로 두고 별도 worktree에서 Claude 띄움. 
claude --worktree experiment/quick-poc
# → .claude/worktrees/experiment-quick-poc 에서 독립 작업
# → 메인 작업 디렉토리는 영향 없음
결과: 실험이 망해도 메인은 안전. 마음에 들면 그 브랜치로 PR.

첫 세션 — 5분 워크플로우

Claude Code의 진가는 프로젝트 컨텍스트를 자동으로 이해한다는 데 있습니다. 다음 흐름을 그대로 따라 해 보세요.

  1. 프로젝트로 이동 — Git 리포지토리 루트가 가장 좋습니다.
  2. claude 실행 — 자동으로 디렉토리 구조를 인지합니다.
  3. /init 실행 — Claude가 코드를 훑고 CLAUDE.md 초안을 만들어 줍니다.
  4. 가벼운 질문 — 예: "이 프로젝트는 뭐 하는 거야?", "build·test 명령은 뭐야?"
  5. 작은 작업 위임 — 예: "README의 오타 수정만 해줘"
cd ~/my-project
claude
# 세션 안에서
> /init
> "이 프로젝트는 뭐 하는 건지 3줄로 설명해줘"
> "package.json을 보고 dev·build·test 명령을 표로 정리해줘"
> "README.md의 한국어 오타만 찾아서 고쳐줘"
⚠️ 주의 — 권한 프롬프트가 나오면 차분히 읽기

처음에는 거의 모든 명령(파일 쓰기, Bash 실행 등)에 권한 프롬프트가 뜹니다. 무조건 Yes를 누르지 말고 어떤 도구를 어떤 인자로 실행하는지 보세요. 익숙해지면 자주 쓰는 명령을 "Yes, don't ask again"으로 화이트리스트에 추가해 갑니다.

핵심 단축키 — 외우면 일이 빨라집니다

모든 키를 외울 필요는 없습니다. 이 6개만 손에 익히세요.

키동작왜 중요한가
Shift+Tab권한 모드 순환 (default → plan → ...)위험 작업 직전에 plan으로 전환
Ctrl+J줄 바꿈 (전송 없음)긴 프롬프트 여러 줄 작성
Ctrl+C현재 작업 중단잘못된 방향이다 싶을 때 즉시 멈춤
Ctrl+L화면 클리어 (대화는 유지)출력이 너무 많을 때 화면만 정리
Ctrl+R이전 프롬프트 히스토리 검색같은 작업 반복 시 빠르게 호출
Esc응답 중 회피·드롭다운 닫기"잠깐, 그거 아냐" 할 때
💡 팁 — 커스텀 단축키

~/.claude/keybindings.json으로 단축키를 본인 취향에 맞게 바꿀 수 있습니다. Vim 사용자라면 스크롤 영역에서 j/k/gg/G를 그대로 쓸 수도 있어요. 자세한 건 /keybindings로 편집기를 열어 보세요.

IDE 통합 — VS Code · JetBrains

터미널보다 IDE 안에서 쓰고 싶다면, Claude Code는 두 IDE를 1급으로 지원합니다.

  • VS Code: Extensions → "Claude Code" 설치 후, 통합 터미널에서 claude 실행. 사이드바 패널·인라인 diff·@-멘션으로 선택 코드 공유 가능.
  • JetBrains (IntelliJ · PyCharm · WebStorm 등): Settings → Plugins → "Claude Code" 설치. Cmd/Alt+Ctrl+K로 파일 참조.
# 이미 IDE를 켠 상태에서 통합 터미널로
claude
# 세션 안에서
/ide
# → 현재 IDE와 연결 (선택 텍스트·diff·진단 자동 공유)

1장 체크리스트

  • ☐ claude --version으로 설치 확인
  • ☐ 첫 claude 세션에서 로그인 완료
  • ☐ 본인 프로젝트에서 /init으로 CLAUDE.md 생성
  • ☐ Plan mode와 일반 모드 Shift+Tab으로 토글해 보기
  • ☐ claude -p "..."로 one-shot 한 번 실행
  • ☐ IDE에서 /ide 연결 (선택)
