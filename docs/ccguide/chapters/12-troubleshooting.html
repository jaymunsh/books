Chapter 12

트러블슈팅 — Claude가 이상하게 행동할 때

가이드를 다 외워도 막상 만나면 당황하는 8가지 상황과 해결법. 강의실에서 가장 자주 받는 SOS만 골랐습니다.

1) 권한 프롬프트가 무한 반복

증상: 같은 명령(npm run test)인데 매번 권한을 묻는다.

원인: 허용 패턴이 너무 좁아서 인자가 다를 때마다 새로 매칭됨. 예: Bash(npm run test)는 정확히 그 명령만 허용하고, npm run test src/api.test.ts는 다시 묻습니다.

해결: 와일드카드로 넓혀 주세요.

{
  "permissions": {
    "allow": [
      // ❌ 정확히 일치만 허용 → 매번 묻기
      "Bash(npm run test)",
      // ✅ 인자 포함 모두 허용
      "Bash(npm run test *)",
      // ✅ 더 넓게 — npm run 자체를 통째로
      "Bash(npm run *)"
    ]
  }
}
💡 빠른 해결

한 세션에서 같은 명령을 3번째 묻고 있다면 "Yes, don't ask again"을 고를 때 옵션 중에 "패턴으로 허용"이 있으면 그쪽을 선택. 자동으로 *가 들어간 규칙이 만들어집니다.

2) 컨텍스트가 폭발 — /compact도 효과 없음

증상: /context로 보니 180k/200k. /compact를 돌렸는데 170k로만 줄고 응답이 점점 느려진다.

원인 진단 순서:

  1. 큰 파일을 통째로 읽었나? 1000줄짜리 파일 하나가 4k+ 토큰입니다.
  2. 이미지/스크린샷이 여러 장 들어 있나? 이미지는 토큰을 크게 잡아먹습니다.
  3. CLAUDE.md가 200줄 넘었나? 매 세션마다 소모됩니다.
  4. 같은 디렉토리를 여러 번 ls·grep했나? 누적됨.

단계별 해결:

# 1단계: 결론·다음 액션만 남기고 정리
> "지금까지 결론과 남은 작업 3가지만 요약해줘. 자세한 분석 과정은 버려도 됨."
> /compact

# 2단계: 그래도 부족하면 새 세션으로
> "위 요약을 다음 세션 시작 프롬프트로 쓸 수 있게 마크다운으로 정리"
> /exit
$ claude     # 새 세션
> [요약 붙여넣기] "이 상태에서 이어가자"

# 3단계: 1M 컨텍스트 모델로 전환
> /model     # 1M 지원 모델로 (Opus 4.7 등)
🎓 강사 노트 — "압축보다 결정적 정리"

/compact는 자동 요약이지만 무엇이 중요한지 모릅니다. 요약을 사람이 직접 시키는 게 훨씬 효과적입니다. "결론과 다음 액션만"이라는 명시적 지시가 핵심.

📌 더 깊이 — 세션 이어가기 4가지 패턴

여기서 다룬 건 "긴급 응급처치" 수준입니다. 며칠~몇 주 짜리 작업을 끊김 없이 이어가는 전략(handoff.md · 일일 작업 일지 · worktree 멀티 세션 등)은 13장 §세션 운영·이어가기 전략에서 자세히 다룹니다. 장기 작업을 한다면 반드시 보세요.

3) 잘못 수정한 파일 복구

증상: Claude가 30개 파일을 수정했는데, 절반은 잘못된 방향이었다.

3단 복구 전략:

상황방법난이도
방금 한 작업만 되돌리기세션 안에서 /undo 또는 "마지막 변경을 되돌려줘"쉬움
여러 단계 전 상태로fileCheckpointing 활성화 상태라면 /checkpoints로 선택 복원중간
git에 한 번도 커밋 안 됨이미 늦음. Claude 응답의 diff 블록을 역으로 적용어려움

예방책: 큰 작업 직전에 항상 git add -A && git commit -m "WIP before claude". 30초 투자로 큰 손실 방지.

# settings.json — checkpointing 명시 활성화
{
  "fileCheckpointingEnabled": true
}
⚠️ git stash는 함정

git stash로 잠깐 치워뒀는데 Claude가 새 파일을 만들면 stash와 충돌할 수 있습니다. commit이 stash보다 안전합니다.

4) 토큰이 갑자기 폭발

증상: 평소 세션은 50k인데 어떤 작업만 200k로 튐.

진단 한 줄:

npx ccusage@latest session --order desc | head -20
# → 가장 비싼 세션 식별

흔한 범인 5가지:

  1. 큰 로그 파일 grep — grep ERROR /var/log/app.log가 10MB 반환
  2. 이미지 첨부 — 스크린샷 4장만 해도 컨텍스트 30% 차지
  3. node_modules 검색 — find . -name "*.ts"가 의존성 포함
  4. 큰 JSON 출력 — cat package-lock.json은 절대 금지
  5. 반복 재시도 — 같은 빌드 명령을 7번 실행

예방 패턴:

# 큰 출력은 head/grep으로 잘라서 주기
> "npm run build 결과 중 'error' 포함 줄만 보여줘"
#  → Claude가 알아서 grep | head 조합

# node_modules·dist 제외 검색 패턴
> "find src -name '*.ts' (node_modules·dist 제외)"

5) "왜 이걸 했지?" — 작업 추적

증상: 며칠 전 Claude한테 시킨 작업의 의도가 기억 안 남. git log를 봐도 모호.

3곳에서 찾기:

소스위치유효 기간
Auto memory~/.claude/projects/<proj>/memory/MEMORY.md영구 (정리하기 전까지)
세션 transcript~/.claude/projects/<proj>/sessions/대부분 유지
Git commit 메시지git log --grep="claude" -p영구
💡 습관 만들기

Claude한테 변경 마무리 시킬 때 "왜 그렇게 했는지 한 줄로 commit message에 포함"을 항상 요청. 한 달 뒤의 본인을 살립니다.

6) 자주 만나는 에러 메시지 해석

메시지의미대처
Permission denied: Bash(...)deny 규칙에 걸림의도된 차단인지 확인. 아니면 settings.json deny 조정
Rate limit exceeded5시간 윈도우 한도 초과잠시 대기 또는 모델 다운그레이드. ccusage로 패턴 확인
Context window exceeded200k 한도 도달위 2번 해결책 또는 1M 모델 전환
Tool execution failed: ...도구(MCP·Bash 등) 실행 자체 실패해당 도구의 stderr 확인 (/mcp로 MCP 상태 점검)
Session not found-c·-r 시 세션 ID 만료claude로 새 세션 시작
Hook exited with non-zero훅 스크립트가 차단해당 훅의 로그 확인. 의도된 거면 무시

/doctor — 막힐 땐 첫 명령

설치·로그인·MCP·hooks의 상태를 한 번에 점검. "뭐가 문제인지 모르겠음"이면 가장 먼저 실행하세요.

> /doctor

# 예시 출력:
#   ✓ claude --version: 2.1.174
#   ✓ Login: OK (user@example.com)
#   ✓ Settings: ~/.claude/settings.json valid
#   ✗ MCP[playwright]: server not responding ← 여기!
#   ✓ Hooks: 3 registered, all executable

최후의 수단 — 새 세션으로 가는 결정 기준

다 시도해도 안 풀리면 새 세션이 답입니다. 아래 셋 중 하나라도 해당되면 미루지 말고 새로.

  • 같은 질문을 3번 했는데 Claude의 답이 모순됨
  • 응답 첫 토큰까지 30초 이상 걸림 (컨텍스트 과부하 신호)
  • "내가 지금 뭘 시키고 있었지?"가 본인부터 헷갈림
# 깔끔하게 인수인계하는 패턴
> "새 세션으로 가려고 한다. 다음 사람(=다음 세션의 나)이 5분 안에 이어 작업할 수 있도록
   현재 상태·다음 액션·주의점을 마크다운으로 요약. 파일 수정 X, 출력만."
> /exit

$ claude
> [요약 붙여넣기] "여기서부터 이어가자"

12장 체크리스트

  • ☐ 권한 패턴에 * 와일드카드 활용 — 무한 반복 방지
  • ☐ fileCheckpointingEnabled: true 설정 확인
  • ☐ 큰 작업 직전 git commit -m "WIP" 습관
  • ☐ ccusage session --order desc로 비싼 세션 식별 1회/주
  • ☐ 문제 생기면 /doctor를 첫 명령으로
  • ☐ 막히면 새 세션 — 위 3가지 신호 기준
