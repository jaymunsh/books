🎓 강사 노트 · 이 가이드를 쓰는 법

Claude Code는 빠르게 진화하고 있습니다. "어디부터 봐야 할지" 모르겠다면, 아래 세 가지 학습 경로 중 본인 위치에 맞는 하나를 골라 시작하세요. 각 경로는 30분 ~ 2시간 안에 끝납니다.

🌱

처음 써본다면 — 30분 코스

  • 1장 설치·로그인·`claude` 첫 실행
  • 2장 `/init`으로 CLAUDE.md 만들기
  • 2장 권한 시스템 이해(allow/deny)
  • 실습: 본인 프로젝트에서 코드 하나 수정
💼

실무에 바로 쓰고 싶다면 — 1시간

  • 1·2장 빠르게 스킵
  • 3장 내장 스킬 (`/code-review`, `/verify`, `/security-review`)
  • 4장 MCP로 GitHub·Slack 연결
  • 5장 헤드리스 모드로 CI에 붙이기
🚀

자동화·확장까지 — 2시간 코스

  • 3장 커스텀 스킬·서브에이전트
  • 5장 `/loop` · `/schedule` 자동화
  • 10장 커뮤니티 도구(superpowers · ccusage 등)
  • 6·7·8장 Workflow · SDK · API (예정)
📖 처음이 막막하다면 — 11장 "실무 빌딩 스토리"로 시작

이론보다 실제 서비스 구축 흐름이 먼저 궁금하다면, 11장의 세 가지 스토리(결혼식 사이트 · URL 단축 SaaS · 가족 PWA)를 먼저 읽으세요. 본인 프로젝트가 어느 패턴에 가까운지 감이 잡힌 뒤 나머지를 읽으면 흡수 속도가 훨씬 빠릅니다.

⚡ 1분 빠른 시작

이미 설치돼 있다면 아래 명령들만 외워도 절반은 끝납니다.

# 설치 (macOS / Linux)
curl -fsSL https://claude.ai/install.sh | bash

# 프로젝트 진입 → 대화형 세션 시작
cd ~/my-project
claude

# 한 번만 실행 (one-shot, 자동화·CI에 유용)
claude -p "이 프로젝트의 README를 200자로 요약해줘"

# 이전 대화 이어 가기
claude -c

# 자주 쓰는 슬래시 명령 (세션 안에서)
#   /init      → CLAUDE.md 자동 생성
#   /model     → 모델 변경 (Opus / Sonnet / Haiku)
#   /agents    → 서브에이전트 관리
#   /mcp       → MCP 서버 추가·확인
#   /compact   → 컨텍스트 압축
# Claude Code를 라이브러리처럼 임베드 — Agent SDK (Python)
# pip install claude-agent-sdk  (※ 패키지명은 공식 문서에서 최종 확인)

from claude_agent_sdk import ClaudeAgent

agent = ClaudeAgent(
    model="claude-sonnet-4-6",
    system_prompt="You are a senior Python reviewer.",
)

result = agent.run("src/api.py의 보안 취약점을 찾아줘")
print(result.text)
// Agent SDK (TypeScript)
// npm i @anthropic-ai/claude-agent-sdk  (※ 패키지명은 공식 문서에서 최종 확인)

import { ClaudeAgent } from "@anthropic-ai/claude-agent-sdk";

const agent = new ClaudeAgent({
  model: "claude-sonnet-4-6",
  systemPrompt: "You are a senior TypeScript reviewer.",
});

const result = await agent.run("src/api.ts의 보안 취약점을 찾아줘");
console.log(result.text);
🎓 강사 노트

처음에는 Bash 탭만 봐도 충분합니다. Claude Code를 그냥 사용한다면 SDK는 필요 없어요. Python·TypeScript SDK는 Claude Code를 본인 서비스에 임베드할 때(예: 슬랙봇, CI 자동화) 등장합니다. 7장에서 다시 만납니다.

🛠 이 가이드를 활용하는 팁

  • 검색 (⌘K / Ctrl+K) — 명령어·키워드를 빠르게 찾을 수 있습니다.
  • 코드 탭 — 모든 예제는 Bash · Python · TypeScript 탭으로 전환 가능합니다(해당되는 경우).
  • 다크 모드 — 우측 상단 토글. OS 설정도 자동 감지합니다.
  • 인쇄 / PDF — 브라우저 인쇄(⌘P)하면 사이드바·복사 버튼이 제거된 책 형태로 출력됩니다.
  • 오프라인 — 외부 의존성 0. 파일을 폴더째 복사해 어디서든 열 수 있습니다.

❓ 시작 전 자주 묻는 질문

도입을 망설이게 하는 질문 7가지. 클릭해서 펼치세요.

Cursor / Copilot / Cline 쓰던 사람도 옮겨오기 쉬운가요?

예, 하지만 사고방식 전환은 필요합니다.

Cursor·Copilot은 주로 IDE 안에서 코드 자동완성·인라인 편집 중심이라면, Claude Code는 터미널 기반 자율 에이전트에 가깝습니다. "탭으로 코드 완성"보다 "이걸 해줘"라고 위임하는 비중이 큽니다.

병행 사용도 가능합니다 — IDE 안 즉시 자동완성은 Copilot, 멀티스텝 작업·리팩터링·디버깅은 Claude Code 식으로 역할 분담하는 패턴도 흔합니다. 14장 §AI 코드 리뷰 가이드라인 참조.

Pro/Max 구독 vs API 직결 — 뭐가 나을까요?

혼자 또는 학습 단계라면 Pro·Max가 마음 편합니다. 매번 토큰 신경 안 써도 되고, 월 정액이라 예산도 예측 가능.

팀·자동화·CI에 붙일 거라면 API 직결 + Anthropic Console이 적합합니다. spending limit·사용량 분석·여러 키 분리가 가능하고, 결제도 회사 계정으로 깔끔.

17장 §팀 규모·예산 가이드라인에 규모별 권장 표가 있어요.

회사 보안 정책상 외부 클라우드 AI를 못 쓰는데요?

방법이 있습니다. Claude Code는 Amazon Bedrock · Google Vertex AI · Microsoft Foundry를 백엔드로 지원합니다. 기존 클라우드 계약·VPC 안에서 호출되므로 보통 회사 보안팀이 받아들이는 경로입니다.

또한 settings.json의 권한·hook으로 어떤 명령·어떤 파일에 접근 가능한지를 코드로 강제할 수 있어 감사 추적도 가능합니다. 16장의 🔴 엔터프라이즈 settings 템플릿 참조.

한국어 응답·한국어 코드 주석 품질은 어떤가요?

Sonnet 4.6·Opus 4.7 모두 한국어 품질이 매우 좋습니다. 영어와 거의 차이가 없는 수준이고, 한국 개발자 관용어(예: "유스케이스", "워크플로우")도 자연스럽게 사용합니다.

다만 코드 주석·문서를 한국어로 쓰게 하려면 명시적으로 지시해야 합니다. CLAUDE.md에 "모든 주석·문서는 한국어로" 한 줄 박아두는 게 가장 확실합니다. settings.json의 "language": "korean"도 응답 언어에 도움이 됩니다.

월 비용이 얼마나 들까요? 감을 잡고 싶습니다

대략적인 벤치마크 (17장에 자세한 표 있음):

  • 주말 사이드 프로젝트 위주: 월 $10~50
  • 풀타임 1인 개발: 월 $50~200
  • 2~5인 스타트업: 월 $200~600
  • 5~20인 팀(CI 포함): 월 $600~2,000

Pro/Max 구독($20~200/월)으로 시작해서 한도가 부족할 때 API 전환을 고려하는 게 안전합니다. npx ccusage@latest로 매주 한 번씩 확인하세요 — 가시화만 해도 사용량이 줄어듭니다.

Claude가 짠 코드를 그대로 머지해도 되나요?

안 됩니다. AI가 짰든 사람이 짰든, PR 작성자가 모든 라인을 이해하고 책임진다는 원칙은 똑같습니다.

실무에서 권장하는 워크플로우:

  1. 본인 셀프 리뷰 — /code-review 한 번 (3장)
  2. 실행 검증 — /verify로 실제 동작 확인
  3. 사람 리뷰어 1명 이상 — 자동 리뷰 봇으로 대체 X

14장 §AI 코드 리뷰 가이드라인에 5가지 합의 사항이 있습니다.

학습 곡선은 어느 정도인가요? 며칠이면 익숙해질까요?

대략적인 단계 (개인차 큼):

  • Day 1 · 30분: 설치·로그인·첫 claude·/init (1장)
  • 1주차: 슬래시 명령·권한·Plan mode 익숙해짐. 작은 작업 위임 가능 (2장)
  • 2~3주차: 스킬·MCP 적극 활용. 본인 워크플로우 구축 (3·4장)
  • 1개월+: 커스텀 스킬·서브에이전트·자동화. "도구가 손에 익은 느낌" (5·11장)

처음부터 다 외울 필요 없습니다. 11장 §스토리 C("균형") 패턴으로 시작 → 필요해질 때 도구 추가, 가 가장 자연스러운 경로입니다.
