# MarketFlow 기획 문서 저장소

PRD(제품 요구사항 문서), 기능 스펙, 릴리즈 노트를 관리하는 저장소입니다.
코드가 아니라 **문서**를 다룹니다.

## 문서 규칙
- 모든 PRD는 `docs/prd/YYYY-MM-DD-기능명.md` 형식으로 저장한다.
- PRD는 반드시 이 순서를 따른다: 배경 → 문제 → 목표(측정 가능한 지표) → 비범위 → 사용자 시나리오 → 요구사항 → 오픈 이슈.
- "빠르게", "직관적으로" 같은 모호한 표현 대신 **측정 가능한 기준**을 쓴다.
  (예: "3초 안에 로딩" / "탭 2번 안에 결제 완료")
- 결정된 사항과 미결 사항을 반드시 구분해 표기한다. 미결은 `⚠️ 미결`로 마킹.

## 작성 원칙
- 사용자가 제어하고 인식하는 언어로 쓴다. 시스템 구현 용어로 쓰지 않는다.
  ("알림 설정"이라 쓰고 "웹훅 config"라 쓰지 않는다)
- 한 문장은 한 가지만 말한다. 복합 요구사항은 항목으로 분리한다.

## 자주 쓰는 작업
- 새 PRD 초안: 위 순서의 골격을 먼저 만들고 섹션별로 채운다.
- 릴리즈 노트: 사용자 관점의 변화만 적는다. 내부 리팩토링은 제외.

## 하지 말 것
- 확정되지 않은 일정·수치를 단정형으로 쓰지 않는다.
- 다른 팀(개발·디자인)이 이미 정한 규칙을 임의로 바꾸지 않는다. 링크로 참조한다.
