# MarketFlow 디자인 시스템 저장소
# 🔀 "다른 팀이라면?" — 디자인팀용 CLAUDE.md 변형 예시

컴포넌트 라이브러리, 디자인 토큰, 아이콘 에셋을 관리하는 저장소입니다.

## 디자인 토큰 규칙
- 색상은 토큰으로만 참조한다. 하드코딩된 hex 금지. (`--color-primary`, `--color-danger` …)
- 간격은 4px 그리드의 배수만 사용한다. (4, 8, 12, 16, 24, 32 …)
- 폰트 크기는 정의된 타입 스케일에서만 고른다. 임의 값 금지.

## 에셋 명명 규칙
- 아이콘: `ic-{이름}-{크기}.svg` (예: `ic-cart-24.svg`)
- 이미지: `img-{맥락}-{이름}@2x.png`
- 컴포넌트 파일: `PascalCase.tsx`, 스토리는 `PascalCase.stories.tsx`

## 접근성 기준 (필수)
- 텍스트 대비는 WCAG AA(4.5:1) 이상. 큰 텍스트는 3:1 이상.
- 모든 인터랙티브 요소는 키보드 포커스가 보여야 한다.
- 아이콘 버튼은 반드시 `aria-label`을 가진다.

## 자주 쓰는 작업
- 새 컴포넌트: 토큰만 사용, 스토리 필수, 접근성 기준 자체 점검 후 완료 보고.
- 아이콘 추가: 24px 그리드에 맞추고 명명 규칙을 따른다.

## 하지 말 것
- 토큰에 없는 값을 컴포넌트에 직접 넣지 않는다. 필요하면 토큰을 먼저 추가한다.
- 브랜드 색을 임의로 변형하지 않는다. 팔레트는 디자인 리드 승인 후 변경.
