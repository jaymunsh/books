# MarketFlow 데이터 저장소

상품·주문·트래픽 분석을 위한 SQL 쿼리, dbt 모델, 리포트 노트북 저장소입니다.

## 데이터 규칙
- 매출 지표는 항상 **부가세 제외(net)** 기준. 세금 포함 값은 `_gross` 접미사를 붙인다.
- 날짜 컬럼은 KST 기준. UTC 원본은 `_utc` 접미사로 구분해 보관한다.
- 개인정보(이메일·전화·주소)는 쿼리 결과에 포함하지 않는다. 필요 시 해시 컬럼을 쓴다.
- 프로덕션 테이블에는 `SELECT`만. `INSERT/UPDATE/DELETE`는 스테이징에서만.

## 자주 쓰는 명령
- 모델 빌드: `dbt build`
- 테스트: `dbt test` — 지표 로직을 바꾸면 반드시 실행한다.
- 쿼리 린트: `sqlfluff lint`

## 지표 정의 (팀 합의)
- **활성 사용자**: 최근 30일 내 1회 이상 주문 또는 장바구니 담기.
- **재구매율**: 첫 구매 후 90일 내 2회차 구매 비율.
- **객단가(AOV)**: net 매출 / 주문 수. 취소·환불 제외.

## 하지 말 것
- 지표 정의를 쿼리마다 새로 만들지 않는다. `models/metrics/`의 정의를 재사용한다.
- 리포트 수치를 근거 쿼리 없이 보고하지 않는다. 항상 재현 가능한 쿼리를 첨부한다.
