# 레거시 결제 모듈 (2024년 이전)

이 폴더(`legacy/`)는 구 결제 시스템입니다. 신규 개발 대상이 아닙니다.
디렉토리 스코프 CLAUDE.md 예시 — 이 폴더 안에서 작업할 때만 아래 규칙이 얹힙니다.

## 이 폴더의 규칙
- 리팩토링·구조 개선을 하지 않는다. 요청된 최소 수정만 한다.
- 새 기능을 여기 추가하지 않는다. 새 결제 기능은 `payments/v2/`에 만든다.
- 버그 수정 시 반드시 기존 테스트를 먼저 실행해 회귀를 확인한다.
- 이 폴더의 코드 스타일은 구식이지만 그대로 둔다. 주변 코드에 맞춘다.

## 왜 이렇게 두는가
- 이 모듈은 곧 폐기 예정이라 개선 투자 가치가 없다.
- 하지만 아직 실서비스가 의존하므로 함부로 건드리면 결제 사고가 난다.
- 그래서 "최소 수정 + 회귀 확인"이 이 폴더의 유일한 작업 방식이다.
