# MarketFlow MCP 연결 예시

`.mcp.json` 은 표준 JSON이라 주석을 넣을 수 없습니다. 각 서버의 의도를 여기 적어 둡니다.

## 서버 설명

- **상품DB** — 상품 데이터베이스를 Postgres MCP 서버로 연결.
  반드시 **읽기 전용(`readonly`) 계정**으로 붙인다. 어떤 실수에도 데이터가 바뀌지 않게.
- **사내위키** — 문서 저장소를 파일시스템 MCP 서버로 노출.
  읽기 범위를 `/docs/wiki` 로 한정해 그 밖을 보지 못하게 한다.
- **이슈트래커** — 원격 HTTP MCP 서버.
  `Authorization` 토큰은 파일에 평문으로 쓰지 말고 환경변수 `ISSUE_TOKEN` 으로 주입한다.

## 보안 체크리스트

- [ ] 읽기만 필요한 서버는 읽기 전용 계정인가?
- [ ] 토큰·비밀번호가 파일에 평문으로 들어가지 않았는가? (`${ENV}` 참조만)
- [ ] 붙이는 서버의 출처가 신뢰할 수 있는가? (사내 감사·승인 목록)
- [ ] 프로덕션과 개발 연결이 분리되어 있는가?

## 실행 전

토큰 환경변수를 먼저 설정한다:

```bash
export ISSUE_TOKEN="..."   # 비밀 저장소에서 주입, 셸 히스토리에 남기지 않기
```
