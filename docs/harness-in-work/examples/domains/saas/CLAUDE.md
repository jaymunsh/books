# TeamSync — B2B SaaS 협업 플랫폼

멀티테넌트 SaaS입니다. 고객사 300곳이 같은 인프라를 공유합니다.
**두 가지가 이 프로젝트의 생명입니다: 테넌트 격리와 하위호환.**

## 테넌트 격리 (최우선)

- 모든 테이블 조회에 `tenant_id` 조건이 **반드시** 들어간다. 예외 없음.
- Repository 계층에서 `tenant_id` 없는 쿼리는 작성하지 않는다.
  전역 조회가 필요하면 `AdminRepository`(별도 승인 필요)를 쓴다.
- 캐시 키에 반드시 `tenant_id`를 포함한다. (테넌트 간 캐시 오염 = 데이터 유출)
- 테스트는 **최소 2개 테넌트**로 작성해 격리를 검증한다.

## API 하위호환

- 공개 API의 응답 필드를 **삭제하거나 이름을 바꾸지 않는다.** 추가만 허용.
- 필드 제거가 필요하면: ① deprecated 표기 → ② 최소 2분기 유지 → ③ 고객 공지 후 제거.
- 필수 요청 파라미터를 새로 추가하지 않는다. 새 파라미터는 항상 선택적 + 기본값.
- 에러 코드는 불변이다. 새 상황은 새 코드를 만든다.
- API 변경 시 `contract-test`가 반드시 통과해야 한다.

## 자주 쓰는 명령
- 테스트: `npm test`
- 계약 테스트(하위호환 검증): `npm run test:contract`
- 테넌트 격리 검증: `npm run test:isolation`
- 타입체크: `npm run typecheck`

## 하지 말 것
- 마이그레이션에서 컬럼 삭제 금지 (다운타임·롤백 불가). 미사용 처리 후 다음 분기에 제거.
