# 한빛물산 ERP 구축 프로젝트 — 도메인 사전

이 저장소는 사내 ERP(회계·재고·구매·인사)를 구축합니다.
**이 파일이 이 프로젝트 하네스의 심장입니다.** 담당자가 바뀌어도 도메인 지식이
유지되도록, 용어와 업무 규칙을 여기에 고정합니다.

## 왜 이 파일이 있는가
ERP는 용어 하나를 잘못 이해하면 회계가 틀어집니다. "전표"와 "증빙"을 혼동하거나
"미결"을 "미승인"으로 처리하면 결산이 깨집니다. 매번 설명하는 대신 여기 고정합니다.

---

## 1. 회계 용어

| 용어 | 코드/필드 | 정의 | 주의 |
|------|-----------|------|------|
| 전표 | `slip` | 회계 거래 1건의 기록 단위 | "증빙(evidence)"과 다름. 증빙은 첨부서류 |
| 미결전표 | `slip.status = PENDING` | 작성됐으나 승인 전인 전표 | "미승인"이 아니라 "미결". 반려는 `REJECTED` |
| 계정과목 | `account_code` | 4자리 숫자 코드 (예: 1010 현금) | 임의 생성 금지. 계정과목표에서만 선택 |
| 차변/대변 | `debit` / `credit` | 복식부기의 좌우 | 합계가 반드시 일치해야 저장 가능 |
| 결산 | `closing` | 월/년 단위 마감 | 마감된 기간의 전표는 수정 불가 |
| 회계기간 | `fiscal_period` | `YYYYMM` 형식 | 회사 회계연도는 1월 시작 |

## 2. 재고 용어

| 용어 | 코드/필드 | 정의 | 주의 |
|------|-----------|------|------|
| 입고 | `receipt` | 창고로 들어온 수량 | 발주(`order`)와 구분. 발주≠입고 |
| 출고 | `issue` | 창고에서 나간 수량 | 판매출고/이동출고를 `issue_type`으로 구분 |
| 실사 | `stocktaking` | 물리적 재고 조사 | 실사 차이는 `adjustment` 전표로 반영 |
| 안전재고 | `safety_stock` | 최소 유지 수량 | 품목별로 다름. 하드코딩 금지 |
| 로트 | `lot_no` | 입고 단위 추적번호 | 식품·의약품 품목은 필수 |

## 3. 구매·인사 용어

| 용어 | 코드/필드 | 정의 |
|------|-----------|------|
| 발주 | `purchase_order` | 거래처에 보낸 주문 |
| 검수 | `inspection` | 입고품 품질 확인. 통과해야 입고 확정 |
| 발령 | `assignment` | 인사 이동 기록. 소급 적용 가능 |
| 급여마감 | `payroll_closing` | 마감 후 급여 데이터 수정 불가 |

---

## 절대 규칙

- **금액은 원화 정수.** 부동소수 금지. 세금 계산은 반드시 `decimal` 사용.
- **차변 합계 = 대변 합계.** 이 검증 없이 전표 저장 로직을 작성하지 않는다.
- **마감된 기간(`closing`, `payroll_closing`)의 데이터는 절대 수정하지 않는다.**
  수정이 필요하면 역분개 전표를 새로 만든다.
- **계정과목·품목코드를 임의로 생성하지 않는다.** 마스터 테이블에서만 조회한다.
- 날짜는 `YYYY-MM-DD`, 회계기간은 `YYYYMM`.

## 하지 말 것

- 프로덕션 DB 직접 수정 금지. 반드시 마이그레이션 + 승인.
