# 레거시 연동 규칙

한빛물산 ERP는 2003년 구축된 AS400 시스템과 당분간 병행 운영합니다.
그 시스템은 수정할 수 없으므로, **우리 쪽이 맞춰야 합니다.**

## 인터페이스 원칙

- 레거시는 **EUC-KR** 인코딩입니다. UTF-8로 읽으면 한글이 깨집니다.
  반드시 경계에서 변환하고, 내부는 UTF-8로 통일합니다.
- 레거시 날짜는 `YYYYMMDD` 문자열입니다. 우리 내부 `LocalDate`로 변환해 씁니다.
- 레거시 금액은 **소수점 없는 문자열**(예: `"0000012345"` = 12,345원)입니다.
  앞자리 0을 살려 고정폭으로 보내야 합니다.
- 레거시 코드값은 우리 코드와 다릅니다. 반드시 `code_mapping` 테이블을 거칩니다.
  직접 매핑을 하드코딩하지 않습니다.

## 배치 연계

- 레거시 배치는 매일 02:00에 돕니다. 그 시간대에 인터페이스 테이블을 건드리지 않습니다.
- 연계 실패 시 **재처리 가능해야** 합니다. 같은 데이터를 두 번 보내도
  중복 생성되지 않도록 `interface_key`로 멱등성을 보장합니다.
- 부분 성공을 허용하지 않습니다. 배치 단위로 전체 성공 또는 전체 롤백.

## 절대 하지 말 것

- 레거시 DB에 직접 INSERT/UPDATE 금지. 반드시 지정된 인터페이스 테이블을 통합니다.
- 레거시 스키마를 변경하려 시도하지 않습니다(권한도 없고 사고가 납니다).
- 인터페이스 규격을 임의로 바꾸지 않습니다. 규격 변경은 양측 합의 문서가 먼저입니다.

## 작업 시 체크리스트

- [ ] 인코딩 변환을 경계에서 처리했는가
- [ ] 금액·날짜 포맷이 레거시 규격에 맞는가
- [ ] 코드 매핑 테이블을 거쳤는가
- [ ] 재처리(멱등성) 시나리오를 테스트했는가
- [ ] 실패 시 롤백이 되는가

참고: 인터페이스 규격서는 `docs/legacy-interface-spec.md`
