요구사항 $ARGUMENTS 의 추적성(traceability)을 검증해줘.

ERP 프로젝트는 감사 대상이라 "요구사항 → 설계 → 구현 → 테스트"가
모두 연결되어야 합니다. 끊긴 고리를 찾는 것이 이 커맨드의 목적입니다.

## 확인 절차

1. **요구사항 정의** — `docs/requirements/` 에서 해당 ID의 정의를 찾는다.
2. **설계 문서** — `docs/design/` 에서 이 요구사항을 참조하는 설계가 있는가.
3. **구현 코드** — 코드 주석·커밋 메시지에서 해당 ID를 검색한다.
   !rg -n "$ARGUMENTS" --type java --type kotlin
4. **테스트** — 이 요구사항을 검증하는 테스트가 존재하는가.
   !rg -n "$ARGUMENTS" --glob '**/test/**'
5. **커밋 이력** — 관련 변경이 언제 들어갔는가.
   !git log --oneline --grep="$ARGUMENTS"

## 출력 형식

| 단계 | 상태 | 위치 |
|------|------|------|
| 요구사항 정의 | ✅/❌ | |
| 설계 문서 | ✅/❌ | |
| 구현 | ✅/❌ | |
| 테스트 | ✅/❌ | |

## 규칙
- 끊긴 고리가 있으면 **명확히 지적**한다. "아마 있을 것"이라 넘어가지 않는다.
- 찾지 못한 것은 "없음"이 아니라 "검색 범위에서 발견되지 않음"으로 보고한다.
- 추측으로 연결하지 않는다. ID가 실제로 명시된 것만 연결로 인정한다.
