# 지표 정의 — 팀 표준

데이터팀 최대의 리스크는 "그럴듯하지만 틀린 수치"입니다.
같은 이름의 지표를 사람마다 다르게 계산하면, 회의에서 숫자가 안 맞습니다.
**정의는 여기 하나뿐입니다. 쿼리마다 새로 만들지 않습니다.**

## 표준 정의

| 지표 | 정의 | 주의 |
|------|------|------|
| 매출(net) | 부가세 **제외**, 취소·환불 차감 후 | 세금 포함은 `_gross` 접미사로 구분 |
| 총매출(gross) | 부가세 포함 | 대외 보고용. 내부 지표는 net 기준 |
| 활성 사용자(MAU) | 최근 30일 내 주문 또는 장바구니 담기 1회 이상 | 단순 로그인은 활성으로 치지 않음 |
| 재구매율 | 첫 구매 후 **90일 내** 2회차 구매 비율 | 기간을 바꾸면 반드시 명시 |
| 객단가(AOV) | net 매출 ÷ 주문 수 | 취소·환불 주문 제외 |
| 이탈률 | 최근 90일 무주문 사용자 비율 | 신규 가입 30일 미만은 모수에서 제외 |

## 계산 규칙

- 기간은 **KST 기준**. UTC 원본은 `_utc` 접미사 컬럼을 쓴다.
- 취소·환불은 **주문일 기준**으로 차감한다(환불일 기준 아님).
- 테스트 계정(`is_test = true`)은 모든 지표에서 제외한다.
- 반올림은 마지막 단계에서 한 번만. 중간 반올림 금지.

## 재현성 (필수)

- **모든 수치에 근거 쿼리를 첨부한다.** 쿼리 없는 숫자는 보고하지 않는다.
- 기존 정의를 재사용한다. `models/metrics/` 의 뷰를 먼저 찾고, 없을 때만 새로 쓴다.
- 지표 정의를 바꿔야 하면 이 파일을 먼저 고치고 팀에 공지한다.
  쿼리에서 임의로 다르게 계산하지 않는다.

## 출력 형식

수치 + 기간 + 근거 쿼리를 함께 낸다.

```
활성 사용자(MAU): 42,310명 (2026-06-01 ~ 06-30, KST)

근거 쿼리:
SELECT COUNT(DISTINCT user_id) ...
```
