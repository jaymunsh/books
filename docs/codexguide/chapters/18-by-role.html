# Codex Cloud로 스크린샷 자동 첨부 (GPT-5-Codex 변형)

AGENTS.md 패턴

## Stack
- React 19 + TS + Vite
- TailwindCSS + shadcn/ui
- TanStack Query + Zustand

## Patterns
- 함수형 + hooks만
- 한 파일 200줄 이내
- API는 src/api/ React Query hook 경유

## A11y
- 모든 button·a·input에 aria
- 키보드 only 가능

## Forbidden
- innerHTML / dangerouslySetInnerHTML
- 직접 fetch

자주 쓰는 워크플로우

  1. 컴포넌트 → 스크린샷 → 다듬기 (Playwright MCP)
  2. Codex Cloud로 디자인 변경 위임 — 스크린샷 자동 첨부
  3. 접근성 자동 점검
🎓 함정

Codex가 만든 UI는 스타일 OK지만 접근성·반응형이 빠지기 쉬움. "키보드 only? 360px 안 깨짐? 다크모드?" 3가지 AGENTS.md에 박기.

AGENTS.md 패턴

## Stack
- Node 22 + TS + Hono
- PostgreSQL 16 + Drizzle
- Redis 7

## Conventions
- 모든 endpoint: Zod 검증 → handler → 응답 헬퍼
- 응답: { ok, data?, error?: { code, message } }
- 트랜잭션: db.transaction(), raw query 금지

## Security
- raw SQL 금지 (Drizzle만)
- 비밀번호 argon2id
- JWT secret은 env만

## Forbidden
- prod DB raw SQL
- console.log (pino만)

워크플로우

  1. 마이그레이션 자동 생성 + Zod 갱신
  2. N+1 발견 — pg MCP로 EXPLAIN
  3. API 변경 후 review profile 체인

AGENTS.md (학습용)

# My Learning Project

## My Level
- 백엔드 1년차, Python 익숙, TS 학습 중

## How to help me
- 코드 줄 때 짧은 설명 함께
- 새 개념엔 한 문단 + 공식 문서 링크
- 내 코드 문제는 "왜"부터 설명
- 모르면 "모른다" (추측 X)

## Project
- 학습 목적, 출시 X
- 안전·완성도보다 "이해" 우선

학습 프롬프트 5가지

  1. 이해 우선: "코드 짜기 전에 접근 2가지 비교 + 추천 + 이유"
  2. "왜?" 묻기: "X 대신 Y 쓴 이유 초보자도 알 수 있게"
  3. 실수 추적: "답 주지 말고 디버깅 시작 힌트 + 어떤 출력 봐야 하는지"
  4. 리뷰 받기: "시니어 관점 리뷰. 스타일 아닌 사고방식 위주"
  5. 새 기술 학습: "5분 안에 핵심 이해 — 해결 문제·차이·핵심 hook·함정"
🎓 가장 빠지기 쉬운 함정

"이해 없이 머지"는 단기엔 빠르지만 장기엔 실력 갉아먹음. 규칙 하나: "PR의 모든 라인을 설명할 수 있는가?" 답이 "아니"면 Codex한테 다시 물어 이해부터 채우세요.

닫는 글 — 변하지 않는 공통점

  • 위험 작업엔 on-request 또는 untrusted approval
  • AGENTS.md는 짧고 구체적 (200줄 이내)
  • 본인 직군 핵심 MCP 1~2개만 신중히
  • 완성 후엔 review profile로 셀프 점검
  • Usage·진척도는 주 1회 확인
