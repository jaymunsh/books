Chapter 09 · 조리법

실전 시나리오 5종 — 따라 하면 동작

앞 장들의 도구를 조합한 실제 구축 가능한 5가지 자동화. 각 시나리오는 준비물·구현 단계·코드·운영 팁으로 정리.

5종 한눈 비교

#시나리오주요 도구구축 시간
1PR 자동 코드 리뷰codex exec + ci profile30분
2대규모 리팩터링Codex Cloud 병렬 위임1~2시간
3지속적 보안 모니터링크론 + Sentry MCP20분
4CI 자체 최적화API + gpt-5.4-mini1시간
5기술 문서 자동화git hooks + Action40분

🥇 시나리오 1 · PR 자동 코드 리뷰

목표: 모든 PR에 5분 안에 1차 리뷰 코멘트 자동 게시.

# .github/workflows/codex-review.yml
name: Codex PR Review
on:
  pull_request:
    types: [opened, synchronize]
jobs:
  review:
    runs-on: ubuntu-latest
    permissions: { contents: read, pull-requests: write }
    steps:
      - uses: actions/checkout@v4
        with: { fetch-depth: 0 }

      - id: size
        run: |
          FILES=$(gh pr view ${{ github.event.number }} --json changedFiles -q .changedFiles)
          if [ "$FILES" -le 5 ]; then echo "effort=low" >> $GITHUB_OUTPUT
          elif [ "$FILES" -le 30 ]; then echo "effort=medium" >> $GITHUB_OUTPUT
          else echo "effort=high" >> $GITHUB_OUTPUT
          fi
        env: { GH_TOKEN: '${{ github.token }}' }

      - run: curl -fsSL https://chatgpt.com/codex/install.sh | sh
      - env:
          OPENAI_API_KEY: ${{ secrets.OPENAI_API_KEY }}
          CODEX_NON_INTERACTIVE: "1"
        run: |
          codex exec --profile ci --full-auto \
            "이 PR diff 리뷰. reasoning_effort=${{ steps.size.outputs.effort }}.
             gh CLI로 인라인 코멘트 게시."
💡 보안 폴더는 무조건 high

auth/·payment/ 변경 PR엔 effort 강제 high. 조건 분기 한 줄 추가.

🥈 시나리오 2 · 대규모 리팩터링 (Codex Cloud 병렬)

목표: 30+ 파일 일괄 변환을 사람 검토 가능한 단위로.

  1. 대상 파일 목록 자동 추출
  2. 모듈별 5개 묶음으로 분할
  3. 각 묶음을 Codex Cloud에 병렬 위임
  4. 5개 PR을 본인이 검토 후 머지
# 1) 대상 식별
> "src/models 의 Sequelize 파일을 5개씩 6묶음으로. 묶음별 복잡도 표."

# 2) Cloud로 6개 위임
> "위 6개 묶음을 각각 Codex Cloud로 위임.
   각 묶음: Sequelize → Drizzle 마이그레이션 + 테스트 통과 + PR 생성.
   PR 제목 'refactor(orm): batch N/6 — <모듈명>'."

# 3) 1~2시간 후 6개 PR 준비 → 검토만
🎓 강사 노트 — Cloud가 정말 필요한가

30파일 미만은 로컬 worktree + 직접 진행이 더 빠를 수 있음. 100+ 파일 또는 동시 진행이 가치 있을 때만 Cloud로.

🥉 시나리오 3 · 지속적 보안 모니터링

목표: 매일 자동 의존성·코드 취약점 스캔 + 슬랙 알림.

# crontab -e
0 6 * * * cd ~/proj && codex exec --profile ci "
1) npm audit --json | jq high·critical
2) src/ 의 인증·검증 누락 점검
3) .env* 파일 노출 여부
종합해 슬랙용 마크다운으로." | slack-cli post --channel "#security-daily"
🎬 보너스 — 자동 패치 PR
0 9 * * 1 cd ~/proj && codex exec --profile ci "
patch/minor 의존성 업데이트만, 각각 별도 PR.
각 PR에서 npm test 통과 확인 후 push."
주의: major 업데이트는 절대 자동화 X — breaking 위험.

🚀 시나리오 4 · CI 파이프라인 자체 최적화

목표: 빌드 로그를 분석해 병목·개선안 제시. gpt-5.4-mini로 비용 절감.

#!/bin/bash
# scripts/analyze-ci.sh

gh run list --limit 10 --json name,conclusion,createdAt,updatedAt \
  | jq '[.[] | {name, conclusion, dur: ((.updatedAt|fromdate) - (.createdAt|fromdate))/60}]' \
  > /tmp/ci.json

curl https://api.openai.com/v1/responses \
  -H "Authorization: Bearer $OPENAI_API_KEY" \
  -H "Content-Type: application/json" \
  -d "$(jq -n --slurpfile data /tmp/ci.json '{
    model: \"gpt-5.4-mini\",
    input: \"이 CI 메트릭 분석:\\n\\($data[0] | @json)\\n1) 가장 느린 2) 변동성 큰 3) 개선 가설 3가지\"
  }')" \
  | jq -r '.output_text' \
  | slack-cli post --channel "#ci-health" --stdin

📚 시나리오 5 · 기술 문서 자동화

목표: 코드 변경 시 README·AGENTS.md·API 문서 최신 상태 유지.

# .github/workflows/docs-sync.yml
on:
  push:
    branches: [main]
    paths: ["src/api/**", "package.json"]
jobs:
  sync:
    runs-on: ubuntu-latest
    permissions: { contents: write, pull-requests: write }
    steps:
      - uses: actions/checkout@v4
      - run: curl -fsSL https://chatgpt.com/codex/install.sh | sh
      - env: { OPENAI_API_KEY: '${{ secrets.OPENAI_API_KEY }}', CODEX_NON_INTERACTIVE: '1' }
        run: |
          codex exec --profile ci "
          최근 머지 변경을 보고 다음 갱신:
          1) README.md Quick Start 섹션 (명령 변화 반영)
          2) docs/api.md 엔드포인트 표
          3) AGENTS.md build/test 명령
          변경 있으면 새 브랜치 docs/auto-sync-{날짜} + PR."
