Chapter 15

비용 최적화 — Usage 너머

같은 작업을 1/3~1/5 비용으로 끝내는 6가지 패턴.

먼저 진단

# Platform Usage 대시보드 가장 정확
# https://platform.openai.com/usage

# 또는 ~/.codex/sessions/ 직접 분석
ls -lhS ~/.codex/sessions/ | head -10  # 큰 세션 순

전형적 진단: 특정 세션 1개가 전체 50%? → 모델 분리 (§1). 매일 비슷 비용? → 캐싱 미적용 (§2). input>>output? → 큰 파일 첨부 안티패턴 (§7).

패턴 1 · 작업별 모델 선택 (가장 큰 레버)

작업권장비용 인덱스*
설계·복잡 디버깅gpt-5.5 + reasoning high5
일반 코딩·리뷰gpt-5.5 (기본)1
분류·변환·요약gpt-5.4-mini0.2
CI 자동 리뷰 (소형)gpt-5.4-mini + low0.15
CI 자동 리뷰 (대형)gpt-5.5 + medium1.2

*gpt-5.5 일반 = 1 기준

패턴 2 · 프롬프트 캐싱 — 자동이지만 의식하면 더 큰 효과

OpenAI는 자동 캐싱. 다만 system/문서를 prefix(앞)에 두고 변경하지 않으면 캐시 매칭이 잘 됨.

# 캐시 히트율 확인
response.usage.input_tokens_details.cached_tokens
# 비율 30%+ 양호, 60%+ 우수

패턴 3 · gpt-5.4-mini 서브에이전트로 비용 격리

# config.toml의 bulk profile
[profiles.bulk]
model = "gpt-5.4-mini"
reasoning_effort = "low"
approval_policy = "never"

codex exec --profile bulk "data/feedback.jsonl 5000건 [Bug/Feature/Other] 분류"

패턴 4 · /compact 적시 사용

사용률액션
~60%그냥
60~80%마무리면 그대로, 길게 가면 1회
80~95%"결론·다음 액션 요약" → compact

패턴 5 · CI 비용 차등

name: Codex Review
on: pull_request
jobs:
  size-check:
    outputs: { tier: ${{ steps.c.outputs.tier }} }
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - id: c
        run: |
          FILES=$(gh pr view ${{ github.event.number }} --json changedFiles -q .changedFiles)
          if [ "$FILES" -le 5 ]; then echo "tier=mini" >> $GITHUB_OUTPUT
          else echo "tier=full" >> $GITHUB_OUTPUT; fi
  review:
    needs: size-check
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: curl -fsSL https://chatgpt.com/codex/install.sh | sh
      - env: { OPENAI_API_KEY: '${{ secrets.OPENAI_API_KEY }}' }
        run: |
          if [ "${{ needs.size-check.outputs.tier }}" = "mini" ]; then
            codex exec --profile ci-mini "리뷰"
          else
            codex exec --profile ci-full "리뷰"
          fi

5분 안에 적용하는 빠른 절감 5가지

  1. 기본 모델 명시 — ~/.codex/config.toml에 model = "gpt-5.5"
  2. AGENTS.md 200줄 이하
  3. 큰 파일 paste 금지 룰
  4. Platform Usage 주간 자동 확인
  5. CI에 effort 차등
