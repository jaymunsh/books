Chapter 01

시작하기 — 설치부터 첫 세션까지

이 장은 Codex CLI를 처음 켜는 순간부터 첫 코드를 수정해 보기까지를 다룹니다. 30분이면 끝나며, 이 장만 마쳐도 일상적 위임 작업은 바로 시작할 수 있습니다.

설치 및 인증

Codex CLI는 터미널에서 실행되는 AI 코딩 에이전트입니다. macOS·Linux·Windows에서 동작합니다.

# Homebrew Cask (가장 안정적)
brew install --cask codex

# 업데이트
brew upgrade codex
# 공식 설치 스크립트
curl -fsSL https://chatgpt.com/codex/install.sh | sh

# 자동(비대화) 설치
CODEX_NON_INTERACTIVE=1 curl -fsSL https://chatgpt.com/codex/install.sh | sh

# 설치 확인
codex --version
# npm 전역 설치
npm install -g @openai/codex

# 업데이트
npm update -g @openai/codex
# PowerShell (관리자 권장)
powershell -ExecutionPolicy ByPass -c "irm https://chatgpt.com/codex/install.ps1 | iex"

# 설치 확인
codex --version

인증 — ChatGPT 로그인 vs API 키

Codex는 두 가지 인증 경로를 제공합니다.

# 첫 실행 → 브라우저 자동 오픈 → ChatGPT 로그인
codex
# → ChatGPT Plus/Pro/Business/Enterprise 구독으로 바로 사용

# 또는 OpenAI API 키 직접 사용
export OPENAI_API_KEY=sk-...
codex
방식장점적합한 사람
ChatGPT 로그인월 정액, 한도 신경 X개인 사용, 학습 단계
OpenAI API 키사용량 분석·키 분리·spending limit팀, 자동화, CI
💡 팁 — 처음엔 ChatGPT Plus/Pro 권장

학습 단계에서는 월 정액이 마음 편합니다. Codex CLI를 ChatGPT 로그인으로 바로 쓸 수 있고, 매번 토큰 신경 안 써도 됨. 본격 자동화·CI가 필요해질 때 API로 전환.

실행 모드 — 한 표로 정리

Codex CLI는 같은 codex 명령에 옵션을 붙여 여러 모드로 동작합니다.

모드실행 방법언제 쓰는가
Interactive (대화형)codex가장 일반적. 대화하면서 작업
Exec (한 번 실행)codex exec "..."한 번만 실행하고 종료. CI·스크립트
Resumecodex resume이전 세션 이어가기 (목록에서 선택)
특정 세션 재개codex resume <session-id>ID로 직접 지정
Auto Modecodex --full-auto승인 없이 자동 진행 (위험, 격리 환경만)
Sandbox 변경codex --sandbox <mode>워크스페이스만 / 전체 접근 등 즉시 변경
Cloud 위임Codex Cloud (3장)백그라운드·병렬 작업, IDE 통합

실제 사용 예

# 평소 작업 — 그냥 진입
cd ~/my-project
codex

# "이 README 200자 요약" → 결과만 stdout
codex exec "README.md를 200자 한국어로 요약"

# 어제 하던 대화 이어가기 (목록 표시 → 선택)
codex resume

# 자동 모드 — 컨테이너·VM 같은 격리 환경에서만 권장
codex --full-auto
🎓 강사 노트 — codex resume를 잘 쓰세요

Codex는 세션 자동 저장이 잘 되어 있어서 codex resume으로 어제 작업을 그대로 이어가기 쉽습니다. "매일 새로 시작"보다 "이어가기"가 훨씬 효율적. 13장에서 세션 운영 전략을 자세히 다룹니다.

첫 세션 — 5분 워크플로우

  1. 프로젝트로 이동 — Git 리포지토리 루트가 가장 좋습니다.
  2. codex 실행 — 자동으로 디렉토리 구조 인지.
  3. /init 실행 — Codex가 코드를 훑고 AGENTS.md 초안 생성.
  4. 가벼운 질문 — "이 프로젝트는 뭐 하는 거야?"
  5. 작은 작업 위임 — "README의 오타 수정만 해줘"
cd ~/my-project
codex
# 세션 안에서
> /init
> "이 프로젝트는 뭐 하는 건지 3줄로 설명해줘"
> "package.json을 보고 dev·build·test 명령을 표로 정리해줘"
> "README.md의 한국어 오타만 찾아서 고쳐줘"
⚠️ 주의 — 첫 sandbox 안내

Codex의 기본 sandbox_mode는 workspace-write입니다. 워크스페이스(현재 디렉토리) 내 파일 읽기·쓰기·로컬 명령은 가능, 네트워크는 기본 차단. 만약 npm install 같은 네트워크 명령이 필요하면 그때마다 approval 요청 받습니다. 첫 세션에는 차분히 읽고 승인하세요. 2장에서 자세히.

핵심 단축키

전부 외울 필요 없습니다. 이 6개만 손에 익히세요.

키동작왜 중요한가
Ctrl+J줄 바꿈 (전송 없음)긴 프롬프트 여러 줄
Ctrl+C현재 작업 중단잘못된 방향이면 즉시 멈춤
Ctrl+L화면 클리어 (대화 유지)출력이 많을 때 화면만 정리
Ctrl+R히스토리 검색같은 작업 빠르게 호출
Esc응답 회피·드롭다운 닫기"잠깐, 그거 아냐" 할 때
Ctrl+D세션 종료깔끔하게 끝내기

IDE 통합 — VS Code · JetBrains · Codex IDE Extension

터미널보다 IDE 안에서 쓰고 싶다면, OpenAI는 IDE Extension을 제공합니다.

  • VS Code: Extensions → "Codex" 검색 → Install. 사이드바 채팅·인라인 diff·Codex Cloud 작업 위임·완료 추적.
  • JetBrains: Marketplace에서 "Codex" 설치. 코드 컨텍스트 공유·진단 통합.
  • 통합 터미널 모드: IDE의 통합 터미널에서 codex 실행도 자연스러움.
🎬 시나리오 · "IDE에서 Codex Cloud로 위임 → 다른 일 → 결과 받기"
상황: 큰 기능 1개 + 작은 버그픽스 1개를 동시에 처리해야 함. 적용: 큰 기능은 IDE에서 우클릭 → "Send to Codex Cloud" → 클라우드에서 백그라운드 작업. 그동안 본인은 IDE에서 버그픽스. 결과: 30분 뒤 Cloud 결과 알림 → IDE에서 diff 검토 → 머지. 시간 압축.

1장 체크리스트

  • ☐ codex --version으로 설치 확인
  • ☐ 첫 codex 세션에서 로그인 완료 (ChatGPT 또는 API)
  • ☐ 본인 프로젝트에서 /init으로 AGENTS.md 생성
  • ☐ codex exec "..."로 한 번 one-shot 실행
  • ☐ codex resume으로 세션 재개 시도
  • ☐ IDE Extension 설치 (선택)
  • ☐ sandbox 기본값(workspace-write) 의미 이해
