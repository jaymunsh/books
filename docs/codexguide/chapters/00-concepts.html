기초 개념 — LLM과 Codex CLI 작동 원리

Codex CLI의 명령어를 외우기 전에 알아두면 좋은 "왜 이렇게 동작하는가"의 개념들. 깊은 ML 지식 없이도 이해 가능한 수준으로, 그러나 실무에서 결정을 내릴 때 직접 도움이 되는 정도까지 정리합니다.

💡 이 장은 안 봐도 1장으로 갈 수 있습니다

당장 손을 움직이고 싶다면 1장 시작하기로 바로 가세요. 다만 "왜 Codex가 가끔 이상해지지?", "왜 비용이 이렇게 나오지?" 같은 의문이 들면 언제든 여기로 돌아오세요.

LLM 동작 원리 — 60초 압축

GPT-5.5는 LLM(Large Language Model)의 한 종류입니다. 핵심만 보면 의외로 단순한 구조예요.

① 텍스트는 토큰으로 쪼개진다

입력 텍스트는 모델이 보기 전에 토큰이라는 작은 조각으로 분해됩니다.

"안녕하세요 Codex"  →  ["안", "녕", "하", "세", "요", " Cod", "ex"]   // 약 7토큰
"Hello Codex"        →  ["Hello", " Cod", "ex"]                // 약 3토큰

실용적 시사점: 한국어가 영어보다 같은 의미에 약 2~3배 토큰이 듭니다. 비용·컨텍스트 한도 모두 불리.

② 모델은 "다음 토큰의 확률"만 예측한다

LLM이 하는 일은 매 순간 "다음에 올 가장 그럴듯한 토큰"을 확률로 계산하는 것뿐입니다. 한 토큰씩 반복해 긴 답변이 만들어집니다.

실용적 시사점:

  • 환각 발생 이유: 모델은 "사실"이 아닌 "그럴듯함"을 예측 — 모르는 영역에서도 자신감 있게 답함
  • 같은 질문에 다른 답: 확률에서 샘플링하므로 항상 같지 않음
  • 긴 답일수록 누적 오류: 한 토큰 실수가 다음에 영향

③ 컨텍스트 윈도우 = 모델이 한 번에 볼 수 있는 토큰

GPT-5.5는 최대 400k 토큰까지 처리합니다(에디션·시기에 따라 변동). 시스템 프롬프트·AGENTS.md·대화 히스토리·첨부 파일이 모두 포함됩니다.

④ Reasoning — 답 전에 "생각"하는 단계

GPT-5 계열은 답을 내기 전에 reasoning 단계를 거칠 수 있습니다. 마치 사람이 어려운 문제에 "잠깐, 정리해 보자" 하는 것과 비슷.

없을 때있을 때
입력 → 즉시 답입력 → 추론(N 토큰) → 답
빠르고 저렴느리지만 복잡 추론 강함
분류·요약·변환에 적합수학·디버깅·설계에 적합

Codex CLI는 어떻게 동작하는가

단계별 자세히

  1. 당신이 메시지 입력 → 터미널에서 엔터
  2. 컨텍스트 조립 — Codex가 자동으로 다음을 합칩니다:
    • 시스템 프롬프트 (Codex의 기본 지시)
    • 당신의 AGENTS.md (+ AGENTS.override.md 우선)
    • 이번 세션의 이전 대화 (필요 시 자동 압축)
    • 당신의 새 메시지
  3. OpenAI API 호출 — 전부 토큰화해서 전송
  4. GPT-5.5가 응답 생성 — 텍스트이거나 도구 호출(tool_call)
  5. 도구 호출 분기 — 3축 검사:
    • approval_policy: 사용자 승인 필요?
    • sandbox_mode: 파일·쉘 접근 허용 범위?
    • network_access: 외부 네트워크 허용?
    • 3가지 다 통과해야 실행
  6. 반복 — 모델이 "끝"이라고 판단하거나 텍스트만 반환할 때까지
🎓 강사 노트 — "3축이 곧 Codex의 안전 모델"

Claude Code의 allow/ask/deny가 도구별 1축이라면, Codex는 3축으로 더 세밀합니다. "도구는 허용하지만 네트워크는 금지", "워크스페이스 안만 쓰기는 가능, 시스템 전역은 금지" 같은 분리가 가능. 2장에서 자세히 다룹니다.

알아두면 도움 되는 4가지 메탈 모델

① "Codex는 매번 처음 보는 사람"

새 세션이 시작되면 Codex는 당신을 모릅니다. 그래서 AGENTS.md·codex resume·handoff.md가 필요합니다.

② "Codex는 본인이 한 일을 같은 세션 안에선 잘 기억"

한 세션 안에선 모든 도구 호출·파일 수정·결정이 컨텍스트에 누적됩니다.

③ "도구는 Codex의 손, 3축은 안전장치"

Codex 자체는 텍스트만 생성합니다. 실제 행위는 도구가 하고, 3축 검사가 매번 막아 줍니다.

④ "토큰 = 비용 = 시간"

모든 입력·출력이 토큰. 토큰이 곧 청구액과 응답 시간이 됩니다.

💡 Codex의 차별점 — Cloud

로컬 CLI도 강하지만, Codex의 진짜 차별점은 Codex Cloud(웹 위임)입니다. IDE에서 작업을 클라우드로 던져두고 계속 다른 일을 하다 결과만 받아오는 패턴이 가능. 3장에서 자세히 다룹니다.

🔤 꼭 알아야 할 LLM 용어 16선

📦 컨텍스트 · 메모리

Context Window 컨텍스트 윈도우

모델이 한 번에 볼 수 있는 입력 토큰의 최대 크기. GPT-5.5는 400k+. 시스템·메모리·대화·첨부가 모두 여기.

KV Cache Key-Value 캐시

트랜스포머가 매 토큰 생성 시 이전 토큰들의 Key·Value 행렬을 다시 계산하지 않도록 저장한 캐시. Prompt Caching의 기반.

Prompt Caching 프롬프트 캐싱

같은 시스템 프롬프트·문서 반복 사용 시 입력 토큰을 캐시해 ~50% 비용 절감. OpenAI는 자동 캐싱 (8장).

Context Pollution 컨텍스트 오염

긴 대화에서 누적된 잘못된 추측·삭제된 결정이 다음 응답을 흐림. 해결: /compact·codex resume·handoff.md.

🧩 토큰 · 출력

Token 토큰

모델이 텍스트를 처리하는 최소 단위. 한국어는 영어의 2~3배가 필요해 비용 증가.

Streaming 스트리밍

완성된 답을 한 번에 받지 않고 토큰이 생성되는 대로 점진 출력. OpenAI API는 stream: true.

Temperature 온도

출력의 무작위성. 0=결정적, 1에 가까울수록 다양. 코드는 0~0.3, 창작은 0.7~1.0.

Stop Token 종료 토큰

모델이 "응답 끝"을 알리는 특수 토큰. stop·stop_sequences로 강제 가능.

🎓 모델 · 학습

Pretraining 사전 학습

웹·책 등 거대 텍스트로 다음 토큰 예측만 학습하는 단계. 모델의 "지식" 형성.

Fine-tuning 파인 튜닝

사전 학습 모델을 특정 도메인 데이터로 추가 학습. OpenAI는 일부 모델에서 fine-tuning API 제공.

RLHF 인간 피드백 강화학습

인간 평가로 모델을 정렬. GPT 시리즈의 핵심 정렬 방법.

Reasoning Effort 추론 강도

GPT-5에서 reasoning 단계의 깊이 조절. low/medium/high 또는 토큰 예산으로.

💡 추론 · 패턴

Zero-shot / Few-shot

예시 없이 / 예시 2~5개를 보여주고 같은 패턴 요구. gpt-5.4-mini 사용 시 few-shot이 큰 효과.

Chain of Thought (CoT)

답 전에 "단계별로 생각해 보자" 식으로 추론 과정을 유도. GPT-5 reasoning의 이론적 기반.

Function Calling 함수 호출

모델이 외부 함수·API를 호출. OpenAI 표준. Codex의 Shell·Read·MCP 모두 이 위에서 동작.

RAG 검색 증강 생성

질문 시점에 외부 DB·문서에서 관련 내용을 검색해 컨텍스트에 함께 넣음. 환각 감소·지식 한계 보완.

🎓 강사 노트 — 외울 게 아니라 만났을 때 다시 찾기

이 16개를 다 외울 필요 없습니다. 실무에서 자주 등장하는 Token · Context · Prompt Caching · Function Calling · Reasoning Effort · RAG 정도만 정의를 말할 수 있으면 충분.

"환각"을 줄이는 4가지 습관

  1. 맥락 충분히 주기 — 추측 여지를 줄이면 환각이 줄어듭니다.
  2. 검증 가능한 형식으로 요청 — "표로", "JSON으로" 요청.
  3. 모르면 모른다고 답하라고 지시 — "확신 없으면 '모릅니다'" 한 줄 추가.
  4. 자체 검증 시키기 — "끝나면 본인이 verify" 마무리.

0장 체크리스트

  • ☐ LLM은 "다음 토큰 확률"을 예측한다는 것을 이해
  • ☐ 한국어가 영어보다 약 2~3배 토큰 든다는 점 인지
  • ☐ Codex의 3축(approval·sandbox·network) 안전 모델을 안다
  • ☐ "도구 호출 → 3축 검사 → 실행 → 다음 응답" 흐름 이해
  • ☐ 4가지 메탈 모델 중 하나라도 직관적으로 잡힘
  • ☐ 용어 16선 중 자주 만날 6개는 정의를 말할 수 있다
  • ☐ Codex Cloud가 Codex만의 차별점이라는 점 인지
