Chapter 07

OpenAI Agents SDK — 본인 서비스에 임베드

Codex CLI가 "터미널에서 사용"이라면, Agents SDK는 "본인 코드에 라이브러리로 임베드"입니다. 슬랙봇·CI 자동화·내부 도구 — 어디든 GPT-5의 추론을 끼워 넣을 수 있습니다.

📌 SDK 변화 안내

OpenAI의 Agents SDK(Python: openai-agents, TS: @openai/agents)는 빠르게 변합니다. 본 장의 API 시그니처는 작성 시점 기준이며, 실 사용 전 공식 저장소의 최신 버전을 확인하세요.

CLI 헤드리스 vs Agents SDK

방식적합 상황예
CLI 헤드리스 (codex exec)쉘에서 호출 가능한 단발성 작업CI PR 리뷰, 크론 보안 스캔
Agents SDK (코드 임베드)본인 서비스 안에서 도구·상태를 프로그램으로 통제Slack봇, 데이터 분석 봇
🎓 강사 노트 — 90%는 헤드리스로 충분

SDK는 쉘로 표현 어려운 통제(도구 동적 선택·세션 상태·실시간 스트리밍)가 필요할 때만 꺼내세요.

설치

pip install openai-agents
export OPENAI_API_KEY=sk-...
npm i @openai/agents
export OPENAI_API_KEY=sk-...

Hello World

from agents import Agent, Runner

agent = Agent(
    name="Reviewer",
    model="gpt-5.5",
    instructions="You are a senior Python reviewer.",
)

result = Runner.run_sync(agent, "src/api.py 의 보안 취약점 3가지")
print(result.final_output)
import { Agent, run } from "@openai/agents";

const agent = new Agent({
  name: "Reviewer",
  model: "gpt-5.5",
  instructions: "You are a senior TS reviewer.",
});

const result = await run(agent, "src/api.ts 의 보안 취약점 3가지");
console.log(result.finalOutput);

도구(Tool) 정의

from agents import Agent, Runner, function_tool

@function_tool
def get_user(user_id: str) -> dict:
    """DB에서 사용자 조회"""
    return db.query("SELECT * FROM users WHERE id = ?", [user_id])

@function_tool
def refund(order_id: str, reason: str) -> dict:
    """주문 환불 (위험)"""
    return stripe.refunds.create(charge=order_id, reason=reason)

agent = Agent(
    name="SupportBot",
    model="gpt-5.5",
    instructions="고객 지원 봇. 환불 시 반드시 확인.",
    tools=[get_user, refund],
)

Agent Handoffs — 멀티 에이전트

Agents SDK의 강점은 여러 에이전트가 서로 작업을 넘기는 패턴을 1급으로 지원합니다.

from agents import Agent, Runner

triage = Agent(
    name="Triage",
    model="gpt-5.4-mini",  # 분류만 — 저렴
    instructions="문의를 [billing / technical / general] 분류 후 적절한 에이전트로 handoff.",
)

billing = Agent(name="Billing", model="gpt-5.5", instructions="결제 전문가")
tech    = Agent(name="Tech",    model="gpt-5.5", instructions="기술 지원")
general = Agent(name="General", model="gpt-5.4-mini", instructions="일반 문의")

triage.handoffs = [billing, tech, general]

result = Runner.run_sync(triage, "카드 결제가 두 번 됐어요")
# → triage가 billing으로 자동 handoff → billing이 처리

실전 시나리오

Slack 봇

from agents import Agent, Runner, function_tool
from slack_bolt import App

@function_tool
def query_metric(metric: str) -> dict:
    return prom.query(metric)

ops_agent = Agent(
    name="OpsBot",
    model="gpt-5.5",
    instructions="SRE 어시스턴트. 메트릭 조회 후 가설 제시.",
    tools=[query_metric],
)

app = App(token=os.environ["SLACK_BOT_TOKEN"])

@app.event("app_mention")
def handle(event, say):
    result = Runner.run_sync(ops_agent, event["text"])
    say(result.final_output)

자주 만나는 함정

함정해결
도구 너무 많아 모델이 헷갈림에이전트당 도구 5~10개 이내, 많으면 handoff로 분할
도구 실패에 무한 재시도max_turns 설정 + 실패는 에러 텍스트로 반환
비밀이 도구 결과로 새어 나감함수 내부에서 마스킹 후 반환
한 세션 응답 너무 길어 컨텍스트 폭발긴 작업 여러 run()으로 분리 + 중간 요약

7장 체크리스트

  • ☐ CLI 헤드리스 vs SDK 선택 — 본인 작업에 적합한 길
  • ☐ 도구는 5~10개 이내, description은 1~2줄
  • ☐ max_turns로 안전 cap
  • ☐ 비밀 정보 마스킹
  • ☐ 단순 분류는 gpt-5.4-mini handoff로 비용 절감
  • ☐ 패키지명·API 공식 문서 최종 확인
