Chapter 17 · 사례집

비용 효율화 사례집 — 절감 스토리와 AI 활용 원칙

15장이 기법이라면 17장은 이야기와 원칙. 절감 사례 4개 + 단계별 가이드 + Codex 무관 AI 활용 8원칙.

📌 사례 안내

커뮤니티에 자주 보고되는 패턴을 합성한 가상 케이스. 본인 환경에 옮기면 비슷한 효과 기대.

절감 사례 4개

사례핵심 액션전후절감
① 스타트업전사 디폴트 GPT-5.5 → 작업별 5.4-mini 분리월 $2,200월 $680-69%
② 오픈소스CI에 변경 크기 차등 (mini vs full)월 $760월 $190-75%
③ 솔로 개발자AGENTS.md 240줄→85줄 다이어트세션 17k세션 5k-70%
④ 데이터팀Codex Cloud → 로컬 Batch API + mini일 $40일 $7-83%

📖 사례 ① · 디폴트 모델 분리로 69% 절감

증상: 시리즈 A 스타트업 12명. 도입 3개월차 월 $2,200.

진단: Platform Usage 보니 모든 사용자가 gpt-5.5만 사용 — 분류·정리 같은 단순 작업도 5.5.

적용: bulk profile (gpt-5.4-mini) + 팀 슬랙 가이드 "단순 작업은 /profile bulk".

결과: 월 $2,200 → $680 (-69%). 품질 차이 거의 없음.

🎓 기본값의 폭력

"아무도 의식하지 않는 기본값"이 가장 큰 비용 누수. 도입 첫 주에 config.toml 같이 읽으세요.

📖 사례 ② · CI 차등화 75% 절감

증상: OSS 라이브러리. 주말 외부 PR 23개에 일 $74 청구.

진단: PR 62%가 5파일 이하 소형인데 워크플로우는 모두 gpt-5.5 + reasoning high.

적용: 변경 크기로 분기 (15장 §CI 비용 차등) — 소형은 gpt-5.4-mini + low.

결과: 월 $760 → $190 (-75%). 리뷰 속도는 오히려 빨라짐.

📖 사례 ③ · AGENTS.md 다이어트 70% 절약

증상: 인디 개발자 M. 세션 시작도 안 했는데 컨텍스트 17k 차 있음.

진단: 1년간 "이것도 적어둬야지" 누적된 AGENTS.md 240줄.

📌 짚고 가기 — AGENTS.md는 자동으로 안 커진다

AGENTS.md는 Codex가 자동 추가하지 않습니다. 사람이 직접 편집해야 합니다. M의 경우처럼 "한 줄씩 추가" 1년이면 240줄 누적되는 것이지, 가만히 둔다고 커지지 않아요.

적용: Codex에게 다이어트 직접 시킴.

> "이 AGENTS.md를 90줄 이내로 다이어트:
   1) 지금도 실제 적용되는 규칙만
   2) 중복 통합
   3) '과거에 막혔던' 회상형 메모 삭제 (필요시 .codex/notes/ 로 분리)
   4) '왜 이 규칙인가' 1줄로 안 떠오르면 삭제 후보로 표시"

결과: 세션 시작 컨텍스트 17k → 5k (-70%). 월 토큰 30% 감소.

📖 사례 ④ · Cloud → 로컬 Batch로 83% 절감

증상: 데이터팀. Codex Cloud로 매일 3000건 피드백 분류 — 일 $40.

진단: Cloud는 인터랙티브 작업에 가치 있지만, 단순 배치 분류는 Batch API가 훨씬 저렴.

적용: 8장의 Batch API + gpt-5.4-mini 조합으로 야간 실행.

결과: 일 $40 → $7 (-83%). 분류 정확도 96% → 94% (실용적 차이 없음).

🔄 프로젝트 단계별 효율 시나리오

1단계 · 설계 (Discovery)

  • gpt-5.5 + reasoning high를 짧고 굵게 (1~2회/일, 1시간)
  • 출력: 아키텍처·ADR·기술 스택. 코드 X
  • 예상 비용: $20~50로 끝남

2단계 · 구축 (Build)

  • gpt-5.5 (기본 effort)가 80%
  • 막힐 때만 effort high
  • 매 PR review profile로 셀프 점검
  • 예상: 1인 풀타임 월 $100~200

3단계 · 출시 (Launch)

  • 보안 점검 한 번 high — "값진 $5"
  • CI에 Codex Action + effort 차등
  • Usage baseline 측정

4단계 · 운영 (Operate)

  • 모니터링 자동화는 gpt-5.4-mini
  • 인시던트는 5.5 + high — 아끼지 마세요
  • 주간 Usage 자동 리포트
  • 분기 AGENTS.md·profile 정리

5단계 · 회고 (Retro)

  • 분기 30분: "지난 분기 비싸게 쓴 패턴 3가지"
  • 안티패턴은 config.toml·AGENTS.md에 룰로 박기

🧭 AI 활용 일반 원칙 8가지 (Codex 무관)

GPT·Claude·Gemini — 어떤 AI 도구를 쓰든 적용되는 원칙.

  1. 도구의 무게 = 작업의 무게. 오타에 reasoning high 호출 X.
  2. "비싼 모델 = 좋은 결과"는 신화. 정형 작업은 mini로 충분.
  3. 사람 시간 vs API 비용 트레이드오프. 월 $200 더 써서 5시간 절약은 이득.
  4. 측정 없는 최적화는 추측. 숫자 먼저, 액션 나중.
  5. 컨텍스트가 곧 비용. "매번 보낼 가치가 있나?"
  6. 캐싱은 무조건 켜라. OpenAI는 자동, 다만 prefix를 변경하지 마라.
  7. 배치·비동기로 분리. 실시간성 자기 강요 X. Batch API 50% 절감.
  8. 모델·벤더에 락인되지 마라. AGENTS.md·프롬프트는 평문 마크다운으로.
🎓 한 줄 요약

"측정하고, 작업에 맞는 도구를 골라, 컨텍스트를 절제하고, 사람 시간으로 환산해서 판단한다."

(D) 절약 가치 = B × C$1,000
ROI = (D − A) / A+400%

🎯 닫는 글

모든 사례·원칙은 한 사고방식으로 수렴: "AI 비용을 절약된 사람 시간으로 환산해서 판단". 월 $200이 비싸 보이지만 5시간 살리면 $1,000짜리 결정. 숫자보다 그 숫자가 어떤 시간을 사고 있는가를 보세요.
