- TypeScript strict

🟡 팀 (50줄)

# [프로젝트명]

## Stack
- React 19 + Vite + Tailwind in `apps/web/`
- Hono + Cloudflare Workers in `apps/api/`
- D1 + Drizzle ORM

## Commands
- Dev: `pnpm dev`
- Test: `pnpm test`
- Deploy: `pnpm deploy`

## Conventions
- TypeScript strict, no `any`
- 함수형 컴포넌트 + hooks
- API 응답: `{ ok, data?, error? }`

## Forbidden
- raw SQL (Drizzle만)
- 환경변수는 wrangler secret
- `apps/api/core/auth.ts` 수정 시 integration test 필수

🔵 오픈소스

# [프로젝트명] — Contributor Guide

## Project
- 한 줄: [무엇]
- License: MIT
- Audience: [누구]

## Setup
- Install: `pnpm install`

💬 자주 쓰는 프롬프트 5종

1. 안전한 큰 작업

> "[작업]. 파일 수정 전에 1) 변경 파일 목록 2) 각 파일에 무엇을 바꿀지 먼저 보여줘. OK 받고 진행."

2. 막힌 곳 가설

> "증상: [...]. 시도: [목록]. 가능한 원인 3가지를 가능성 순으로, 각각 검증 방법 한 줄씩."

3. 다 끝낸 척 방지

> "수정 끝나면: 1) 영향 테스트 목록 2) 직접 실행 결과 3) 빠뜨린 케이스 자가 검토"

4. 모호한 결정 되묻기

> "[작업]. 진행 전 확실하지 않은 결정이 있으면 먼저 질문 3개 이내."

5. 인수인계

> "이어가려고 한다. 다음 세션의 나(=너)가 5분 안에 이어 작업할 수 있도록 현재 상태·다음 액션·주의점을 마크다운으로. 파일 수정 X."
