Chapter 13

프롬프팅 기술 + 세션 운영

같은 도구라도 지시 방식으로 결과 품질이 2배 이상 차이. 그리고 며칠짜리 작업을 끊김 없이 이어가는 세션 운영 전략까지.

좋은 vs 나쁜 프롬프트

나쁨좋음
"버그 고쳐줘""POST /users에 빈 email → 500이어야 하는데 200. 어느 케이스 검증 누락?"
"리팩터링해줘""src/auth/login.ts 함수 80줄 → 책임 단위로 3개로. 시그니처 유지, 테스트 그린 유지"
"테스트 작성""src/utils/date.ts의 formatDate에 Vitest unit test. 케이스: 정상·null·잘못된 형식·timezone 경계"
🎓 5요소 공식

① 무엇을 ② 어떻게 ③ 어디까지 ④ 어떤 형태로 ⑤ 하지 말 것.

패턴 1 · "계획부터"

> "... [작업] ... 파일 수정 전에 1) 변경 파일 목록 2) 각 파일에 무엇을 바꿀지 먼저 보여줘. OK 받고 진행."

패턴 2 · GPT-5 reasoning effort

Codex는 작업 복잡도에 따라 reasoning을 조절.

> "단순 문서 정리야. 빠르게."           # → 모델이 자동으로 low 선택
> "이 race condition을 단계별로 깊이 분석"  # → 자동으로 high

# config.toml로 명시
reasoning_effort = "medium"  # 기본

패턴 3 · 컨텍스트 주는 3가지 방법

방법예특징
경로"src/api.ts를 보고..."가장 가벼움
@-멘션 (IDE)@src/api.ts:42-78줄 범위 정확
본문 paste코드 직접가장 비쌈. 외부 데이터만

패턴 4 · 출력 형식 강제

> "package.json deps를 [이름|현재|최신|breaking?] 4컬럼 표"
> "위 결과를 JSON만 출력, 설명 X"
> "보안 점검 결과를 [심각도][파일:줄][제안] 체크박스 마크다운"

패턴 5 · 모델별 프롬프팅

모델스타일
gpt-5.5구체적 지시 + 적당한 여백. 5요소 공식 적용
gpt-5.4-mini매우 명확한 지시 + 예시 1~2개. 모호함 금물
gpt-5.3-codex-spark짧고 빠른 이터레이션. 길게 묻지 말 것

패턴 6 · 부정문 → 긍정문

// ❌ "라이브러리 추가하지 마"
// ✅ "표준 라이브러리만. 의존성 추가 시 먼저 물어보고."

패턴 7 · 자체 검증 요청

> "수정 끝나면: 1) 영향 테스트 목록 2) 직접 실행 결과 3) 빠뜨린 케이스 자가 검토"

세션 운영·이어가기 전략

컨텍스트 사용률별 액션

사용률액션
~50%그냥 진행
50~70%마무리 페이스 빠르게
70~85%/compact 또는 결론 요약
85%+handoff 후 codex resume 또는 새 세션

이어가기 4가지 패턴

① codex resume — Codex가 가장 잘하는 부분

Codex는 세션 자동 저장이 잘 되어 있어서 가장 쉬운 방법.

codex resume          # 목록에서 선택
codex resume <id>     # ID 직접

② /compact — 자동 압축

> /compact            # 30~50% 절감, 핵심 보존

③ 명시적 요약 (자동보다 정확)

> "결론·다음 액션 3가지만 요약. 시도해본 잘못된 가설은 한 줄로만."
> /compact

④ handoff.md — 며칠~몇 주짜리 작업

# 세션 끝낼 때
> ".codex/handoff/2026-06-23-payment.md 로 저장:
   ## 목표
   ## 현재 진행 상태
   ## 시도했지만 안 통한 것
   ## 다음 액션 1~3순위
   ## 주의점"

# 새 세션 시작
$ codex
> "@.codex/handoff/2026-06-23-payment.md 읽고 이어 작업. '다음 액션 1순위'부터."
💡 handoff.md 운영 팁
  • 위치: .codex/handoff/YYYY-MM-DD-주제.md
  • 혼자면 .gitignore, 팀이면 ✅ 커밋
  • 완료된 건 _archive/로 이동
  • 100줄 넘으면 너무 자세함

일일 작업 일지 — 장기 프로젝트

> ".codex/log/2026-06-23.md 작성:
   ## 오늘 한 것
   ## 발견·결정
   ## 내일 시작점
   ## 미해결 질문"

5가지 세션 운영 프롬프트

  1. 시작: "@.codex/handoff/LATEST.md 읽고 현재 상태 요약 후 다음 액션 1순위부터"
  2. 중간 점검: "지금까지 진행 5줄 요약. 다음 할 일은? 막힌 곳?"
  3. 방향 전환: "앞서 X로 가기로 했지만 Y가 낫다. 이전 결정 무효, Y 관점에서 재정리"
  4. 마무리: "오늘 작업 마무리. handoff 파일 작성."
  5. 잘못된 방향 끊기: "3번 시도 모두 같은 패턴 실패. 멈추고: 1) 공통 원인 가설 2) 다른 접근 2가지"

3대 안티패턴

안티왜대신
"잘 만들어줘"기준 없음"이 3가지 만족: ①... ②... ③..."
긴 대화에 새 주제 끼우기이전 컨텍스트 오염새 세션
오류 메시지 통째 paste만맥락 없음"시도한 것 + stderr + 가설 3개"

13장 체크리스트

  • ☐ 큰 작업 전 "계획부터" 한 줄
  • ☐ 파일 참조는 본문 paste 대신 경로/멘션
  • ☐ 출력 형식(표·JSON) 명시
  • ☐ gpt-5.4-mini 사용 시 예시 1~2개 포함
  • ☐ codex resume을 일상화
  • ☐ 장기 작업은 handoff.md로
  • ☐ "잘 만들어줘" 같은 추상어 추방
