Chapter 05

자동화 · CI/CD — 사람 없이 돌게 만들기

Codex의 마지막 도약은 "사람이 옆에 없을 때도 돌게 만드는 것"입니다. codex exec 헤드리스 모드, GitHub Actions 통합, Codex Cloud 자동 위임, ci permission profile까지 — 네 가지 도구가 핵심입니다.

헤드리스 모드 — codex exec

codex exec는 상호작용 없이 stdout으로만 결과를 뱉습니다. CI·크론·쉘 스크립트 어디에서나 호출할 수 있어요.

# 가장 단순한 형태 — 질문 → 답변만 출력
codex exec "package.json의 dependencies를 알파벳 순으로 정렬한 JSON만 출력"

# 권한 자동 승인 — CI 환경에서 필수
codex exec --profile ci --full-auto "src/ 코드 리뷰 후 결과 마크다운"

# 출력을 파일로
codex exec "보안 점검" > security.md

# JSON 출력 (자동화 파이프라인)
codex exec --output json "이슈 분류" | jq .

GitHub Actions — PR 자동 리뷰

가장 흔한 자동화 시나리오. PR이 열리면 Codex가 자동으로 리뷰 코멘트 게시.

# .github/workflows/codex-review.yml
name: Codex PR Review

on:
  pull_request:
    types: [opened, synchronize]

jobs:
  review:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      pull-requests: write
    steps:
      - uses: actions/checkout@v4
        with: { fetch-depth: 0 }

      - name: Install Codex CLI
        run: |
          curl -fsSL https://chatgpt.com/codex/install.sh | sh
          echo "$HOME/.local/bin" >> $GITHUB_PATH
        env:
          CODEX_NON_INTERACTIVE: "1"

      - name: Run review
        env:
          OPENAI_API_KEY: ${{ secrets.OPENAI_API_KEY }}
        run: |
          codex exec --profile ci --full-auto \
            "이 PR의 diff를 리뷰하고 발견 사항을 인라인 코멘트로 게시.
             gh CLI 사용 가능. 카테고리: bug · security · style · suggestion."

GitLab CI 예제

# .gitlab-ci.yml
codex-review:
  stage: review
  image: node:20
  before_script:
    - curl -fsSL https://chatgpt.com/codex/install.sh | sh
    - export PATH="$HOME/.local/bin:$PATH"
  script:
    - codex exec --profile ci --full-auto "merge request 리뷰"
  only:
    - merge_requests
  variables:
    OPENAI_API_KEY: $CI_OPENAI_KEY
    CODEX_NON_INTERACTIVE: "1"
⚠️ CI에서 권한·비용 주의
  • CI는 비대화식 → --profile ci --full-auto 필수, 또는 approval_policy = "never"
  • 외부 컨트리뷰터 PR엔 조건 분기 (악용 방지) — if: github.event.pull_request.author_association != 'NONE'
  • 큰 PR에 매번 gpt-5.5는 비쌈. reasoning_effort = "low" 또는 gpt-5.4-mini로 차등

Codex Cloud — 이슈 자동 처리

Codex Cloud(3장)와 GitHub 통합을 합치면 "이슈 던지면 PR이 돌아오는" 워크플로우가 가능.

# GitHub Issue 본문 또는 코멘트에 멘션

@codex 이 이슈를 분석하고 수정 PR을 만들어줘.
재현 절차:
1. ...
2. ...
기대: ...
실제: ...

15분~1시간 안에 Cloud가 자동으로:

  1. 이슈 분석 → 가설 수립
  2. 브랜치 생성 → 수정 → 테스트
  3. PR 생성 + 본인 멘션
🎬 시나리오 · "작은 버그 10개 → 하룻밤 사이 처리"
상황: 작은 버그 10개가 백로그에 쌓임. 하나씩 처리하기 부담. 적용: 각 이슈에 @codex 고쳐줘 멘션. 본인은 퇴근. 결과: 아침에 10개 PR 준비됨. 리뷰만 하면 끝.

정기 자동 작업 — 크론으로

매일·매주 반복 작업은 OS 크론에 Codex 명령을 박아 둡니다.

# crontab -e

# 매일 새벽 6시 보안 점검
0 6 * * * cd ~/proj && codex exec --profile ci "보안 점검 + 결과 슬랙 전송"

# 매주 월요일 09시 의존성 업데이트 체크
0 9 * * 1 cd ~/proj && codex exec --profile ci "npm outdated 결과 정리 + 슬랙"

# 매시간 CI 상태 모니터링
0 * * * * codex exec "gh run list --limit 5 + 실패 분석" | slack-cli post --channel "#ci-health"

실전 자동화 레시피 3가지

레시피 1 · 외부 PR 자동 1차 리뷰

# .github/workflows/external-pr.yml
on:
  pull_request_target:
    types: [opened, synchronize]
jobs:
  first-pass:
    if: github.event.pull_request.author_association != 'MEMBER'
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with: { ref: '${{ github.event.pull_request.head.sha }}' }
      - run: curl -fsSL https://chatgpt.com/codex/install.sh | sh
      - env: { OPENAI_API_KEY: '${{ secrets.OPENAI_API_KEY }}', CODEX_NON_INTERACTIVE: '1' }
        run: codex exec --profile ci "외부 PR 1차 리뷰 — 명백한 버그·보안만"

레시피 2 · 매일 빌드 분석

#!/bin/bash
# scripts/daily-build-analysis.sh

# 최근 빌드 메트릭 수집
gh run list --limit 20 --json name,conclusion,createdAt,updatedAt > /tmp/ci.json

# Codex로 분석 → 슬랙
codex exec --profile ci "
/tmp/ci.json 의 빌드 데이터에서:
1) 가장 느린 워크플로우
2) 변동성 큰 워크플로우
3) 개선 가설 3가지
마크다운으로 출력." | slack-cli post --channel "#ci-health"

레시피 3 · 머지 후 문서 동기화

# .github/workflows/docs-sync.yml
on:
  push:
    branches: [main]
    paths: ["src/api/**", "package.json"]
jobs:
  sync:
    runs-on: ubuntu-latest
    permissions: { contents: write, pull-requests: write }
    steps:
      - uses: actions/checkout@v4
      - run: curl -fsSL https://chatgpt.com/codex/install.sh | sh
      - env: { OPENAI_API_KEY: '${{ secrets.OPENAI_API_KEY }}', CODEX_NON_INTERACTIVE: '1' }
        run: |
          codex exec --profile ci "
          최근 머지 변경을 보고 README·docs/api.md·AGENTS.md 의 명령 부분 갱신.
          변경 있으면 docs/auto-sync-$(date +%Y%m%d) 브랜치로 PR."

자동화 패턴 한눈에

상황도구예
CI/CD에서 호출codex exec --profile ciPR마다 자동 리뷰

외부에서 멘션Slack 통합 (커뮤니티 도구 10장)슬랙에서 운영 작업

5장 체크리스트

  • ☐ codex exec "..." 한 번 실행해 보기
  • ☐ --profile ci 정의 후 CI 워크플로우 등록
  • ☐ 외부 PR 자동 리뷰 (조건 분기 포함)
  • ☐ GitHub 이슈에 @codex 멘션 → Cloud 자동 처리 시도
  • ☐ 크론으로 일일 보안 점검 자동화
  • ☐ CODEX_NON_INTERACTIVE=1 환경변수 CI에 설정
