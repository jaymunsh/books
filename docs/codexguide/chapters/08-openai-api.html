OpenAI API — 모델을 직접 호출하기

Codex CLI·Agents SDK 너머 — GPT-5를 가장 낮은 수준에서 직접 호출하는 방법. Responses API, 자동 캐싱, Reasoning, Function Calling, Batch·Files API까지.

Responses API — 권장 호출 방식

OpenAI는 새로운 작업엔 Responses API(client.responses.create)를 권장합니다. 이전의 Chat Completions(client.chat.completions.create)와 비교해 도구·상태·reasoning 통합이 더 간결합니다.

from openai import OpenAI

client = OpenAI()

response = client.responses.create(
    model="gpt-5.5",
    input="Python에서 데코레이터를 설명해줘",
)
print(response.output_text)
import OpenAI from "openai";

const client = new OpenAI();

const response = await client.responses.create({
  model: "gpt-5.5",
  input: "TypeScript에서 제네릭을 설명해줘",
});
console.log(response.output_text);
curl https://api.openai.com/v1/responses \
  -H "Authorization: Bearer $OPENAI_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-5.5",
    "input": "Rust에서 ownership을 설명해줘"
  }'

프롬프트 캐싱 — 자동 + 50% 절감

OpenAI는 자동 캐싱을 제공합니다(별도 활성화 불필요). 같은 프롬프트 시작 부분(prefix)이 반복되면 자동으로 캐시 토큰 단가 적용(약 50% 저렴).

캐시 친화 3가지 습관

  1. System 메시지·문서를 prefix에 배치 — 변경되지 않는 부분이 앞에 와야 캐시 매칭
  2. 5분 이내 후속 요청 — 캐시 TTL
  3. 큰 컨텍스트는 같은 세션에서
# 캐시 히트율 측정
response = client.responses.create(...)
print(response.usage.input_tokens_details.cached_tokens)
# 두 번째부터 cached_tokens가 대부분이면 성공

Reasoning — GPT-5의 추론 단계

복잡 추론에 reasoning 강도를 명시. 단순 작업엔 끄고, 복잡 디버깅·수학·아키텍처엔 high.

response = client.responses.create(
    model="gpt-5.5",
    input="이 코드의 race condition을 단계별로 분석",
    reasoning={ "effort": "high" },  # low / medium / high
)
⚠️ Reasoning 토큰도 비용

reasoning effort high는 응답 토큰을 크게 증가시킵니다. 단순 작업엔 끄세요.

Function Calling — 도구 호출

tools = [{
    "type": "function",
    "name": "get_weather",
    "description": "도시의 현재 날씨",
    "parameters": {
        "type": "object",
        "properties": { "city": { "type": "string" } },
        "required": ["city"]
    }
}]

response = client.responses.create(
    model="gpt-5.5",
    input="서울 날씨 알려줘",
    tools=tools,
)

# tool_call 발견 → 실행 → 결과 다시 전달
for item in response.output:
    if item.type == "function_call":
        result = my_get_weather(item.arguments["city"])
        # input에 function_call_output 추가 후 재호출

MCP를 API 클라이언트로 사용

Responses API는 MCP 서버 등록도 지원합니다(SDK가 자동 라우팅).

response = client.responses.create(
    model="gpt-5.5",
    input="지난 1시간 에러 톱 5",
    tools=[{
        "type": "mcp",
        "server_label": "sentry",
        "server_url": "https://mcp.sentry.dev/mcp",
        "headers": { "Authorization": f"Bearer {SENTRY_TOKEN}" },
    }],
)

Files API — PDF·이미지·문서

# 1) 업로드
upload = client.files.create(file=open("manual.pdf", "rb"), purpose="user_data")

# 2) 메시지에서 file_id로 참조
response = client.responses.create(
    model="gpt-5.5",
    input=[
        { "type": "input_file", "file_id": upload.id },
        { "type": "input_text", "text": "이 매뉴얼 핵심을 5줄로" },
    ],
)

Batch API — 50% 절감 대량 처리

"지금 당장" 필요 없는 작업은 Batch로 보내면 비용 절반. 보통 24h 이내 완료.

# 1) JSONL 파일로 요청 준비
with open("batch.jsonl", "w") as f:
    for i, text in enumerate(feedbacks):
        f.write(json.dumps({
            "custom_id": f"feedback-{i}",
            "method": "POST",
            "url": "/v1/responses",
            "body": { "model": "gpt-5.4-mini", "input": f"분류: {text}" }
        }) + "\n")

# 2) 업로드 + 배치 생성
input_file = client.files.create(file=open("batch.jsonl", "rb"), purpose="batch")
batch = client.batches.create(input_file_id=input_file.id, endpoint="/v1/responses", completion_window="24h")

# 3) 폴링 후 결과 다운로드
while client.batches.retrieve(batch.id).status != "completed":
    time.sleep(60)

output = client.files.content(batch.output_file_id)

API 키·청구 안전

  • 키 분리: dev/staging/prod 별도 키
  • 환경변수만: 코드·AGENTS.md·Git에 키 박지 말 것
  • Usage limit: OpenAI Platform → Settings → Limits → 월 한도 + 알림
  • 로그: 모든 호출의 request_id·model·tokens 기록
  • 레이트 리밋: 429 → 지수 백오프 (SDK 자동)

8장 체크리스트

  • ☐ 본인 작업에 적정 모델 선정
  • ☐ 반복 호출은 system prefix를 변경 없이 유지 (자동 캐싱)
  • ☐ 대량 처리는 Batch API + mini 검토
  • ☐ 큰 파일은 Files API + file_id 참조
  • ☐ API 키 환경변수 + dev/prod 분리
  • ☐ Platform에서 spending limit 설정
  • ☐ 새 코드는 Responses API 사용 (Chat Completions 대신)
