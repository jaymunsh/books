⚡ 1분 빠른 시작

이미 설치돼 있다면 아래 명령들만 외워도 절반은 끝납니다.

# 설치 (macOS / Linux)
brew install --cask codex
# 또는
npm install -g @openai/codex

# 프로젝트 진입 → 대화형 세션 시작
cd ~/my-project
codex

# 한 번만 실행 (자동화·CI에 유용)
codex exec "이 프로젝트의 README를 200자로 요약해줘"

# 이전 세션 이어 가기
codex resume

# 자주 쓰는 슬래시 명령 (세션 안에서)
#   /init        → AGENTS.md 자동 생성
#   /model       → 모델 변경 (gpt-5.5 / mini / spark)
#   /approvals   → 권한 모드 변경
#   /mcp         → MCP 서버 추가·확인
#   /compact     → 컨텍스트 압축
# OpenAI API · GPT-5.5 직접 호출 — Python
# pip install openai

from openai import OpenAI

client = OpenAI()

response = client.responses.create(
    model="gpt-5.5",
    input="src/api.py의 보안 취약점을 찾아줘",
)
print(response.output_text)
// OpenAI API · GPT-5.5 직접 호출 — TypeScript
// npm i openai

import OpenAI from "openai";

const client = new OpenAI();

const response = await client.responses.create({
  model: "gpt-5.5",
  input: "src/api.ts의 보안 취약점을 찾아줘",
});

console.log(response.output_text);
🎓 강사 노트

처음에는 Bash 탭만 봐도 충분합니다. Codex CLI를 그냥 사용한다면 SDK는 필요 없어요. Python·TypeScript SDK는 OpenAI 모델을 본인 서비스에 임베드할 때(예: 슬랙봇, CI 자동화) 등장합니다. 7·8장에서 다시 만납니다.

Claude Code와 Codex CLI, 뭐가 다르고 뭐가 같나요?

컨셉은 비슷, 디테일은 다릅니다.

같은 점: 둘 다 터미널 자율 에이전트 · MCP 지원 · 영구 메모리 파일 · 권한 시스템 · 헤드리스 CI 통합 가능.

다른 점:

  • 모델 백엔드 — OpenAI (GPT-5.x) vs Anthropic (Claude 4.x)
  • 메모리 파일 — AGENTS.md (+ .override.md) vs CLAUDE.md
  • 권한 모델 — 3축(approval·sandbox·network) vs allow/ask/deny
  • 클라우드 — Codex Cloud(본격적 웹 위임) vs (Slack 통합 정도)
  • 설정 파일 — config.toml vs settings.json

비교 표는 16장 부록에 있습니다. 병행 사용도 흔합니다(역할 분담).

ChatGPT 구독 vs API 직결 — 뭐가 나을까요?

혼자 또는 학습 단계라면 ChatGPT Plus/Pro가 마음 편합니다. Codex CLI를 ChatGPT 로그인으로 바로 쓸 수 있고, 매번 토큰 신경 안 써도 됨.

팀·자동화·CI에 붙일 거라면 OpenAI API 직결이 적합. 사용량 분석·여러 키 분리·spending limit 모두 가능.

15장 §비용 최적화에 규모별 권장 표가 있어요.

회사 보안 정책상 외부 클라우드 AI를 못 쓰는데요?

방법이 있습니다. OpenAI는 Azure OpenAI Service를 통해 동일 모델을 제공하며, 기존 Azure 계약·VPC 안에서 호출됩니다. 보통 보안팀이 받아들이는 경로.

또한 Codex의 sandbox 모드는 기본이 workspace-write로 워크스페이스 외부 접근을 막아주는 OS-level 격리를 제공합니다.

한국어 응답·한국어 코드 주석 품질은 어떤가요?

GPT-5.5는 한국어 품질이 우수합니다. 한국 개발자 관용어, 자연스러운 코멘트 모두 양호.

다만 코드 주석·문서를 한국어로 쓰게 하려면 명시적으로 지시해야 합니다. AGENTS.md에 "모든 주석·문서는 한국어로" 한 줄 박아두는 게 가장 확실합니다.

월 비용이 얼마나 들까요? 감을 잡고 싶습니다

대략적인 벤치마크 (17장에 자세한 표 있음):

  • 주말 사이드 프로젝트: 월 $10~50
  • 풀타임 1인 개발: 월 $50~200
  • 2~5인 스타트업: 월 $200~600
  • 5~20인 팀(CI 포함): 월 $600~2,000

ChatGPT Plus($20)·Pro($200)로 시작해서 한도 부족 시 API 전환을 고려하는 게 안전합니다.

Codex가 짠 코드를 그대로 머지해도 되나요?

안 됩니다. AI가 짰든 사람이 짰든 PR 작성자가 모든 라인을 이해하고 책임진다는 원칙은 똑같습니다.

워크플로우 권장:

  1. 본인 셀프 리뷰 — diff 직접 훑기
  2. 실행 검증 — 실제 동작 확인
  3. 사람 리뷰어 1명 이상 — 자동 봇이 봤어도 사람 1명은 필수
학습 곡선은 어느 정도인가요?

대략적인 단계:

  • Day 1 · 30분: 설치·로그인·첫 codex·/init
  • 1주차: 슬래시 명령·sandbox·approval 익숙해짐
  • 2~3주차: Codex Cloud·MCP·자동화 패턴 구축
  • 1개월+: permission profile 공유·CI 통합·"손에 익은 느낌"

처음부터 다 외울 필요 없습니다. 11장 §스토리 C("균형") 패턴으로 시작 → 필요해질 때 도구 추가가 가장 자연스러운 경로입니다.

🛠 이 가이드를 활용하는 팁

  • 검색 (⌘K / Ctrl+K) — 명령어·키워드를 빠르게 찾을 수 있습니다.
  • 코드 탭 — 모든 예제는 Bash · Python · TypeScript 탭으로 전환 가능합니다(해당되는 경우).
  • 다크 모드 — 우측 상단 토글. OS 설정도 자동 감지합니다.
  • 인쇄 / PDF — 브라우저 인쇄(⌘P)하면 사이드바·복사 버튼이 제거된 책 형태로 출력됩니다.
  • 진척도 — 각 챕터 끝 체크리스트는 localStorage에 자동 저장됩니다.
